# Спека: UI cleanup hero и related

## Проблемы

1. На главной hero-подзаголовок местами визуально сливается с фоном.
2. В статье блок `Читать также:` дублируется:
   - сначала текстовой строкой внутри body;
   - затем полноценным related-блоком карточек внизу.

## Цель

Убрать визуальную поломку и текстовое дублирование, не ломая текущую editorial-систему и не ухудшая SEO/перелинковку.

## Что сделать

### 1. Hero на главной

**Файл:** `app/page.tsx`

- Проверить реальный итоговый цвет подзаголовка в обеих темах.
- Если проблема вызвана токеном `text-muted`, не патчить инлайн-цветом, а подобрать корректный токеновый класс или отдельный semantic utility в рамках design system.
- Сохранить текущую типографику hero: serif H1 + muted supporting line.
- Проверить контраст не только в light, но и в dark.

### 2. Удалить дублирующий текстовый related внутри статьи

**Файл:** `app/articles/[slug]/page.tsx`

- Убрать inline-абзац `Читать также:` из блока основного текста.
- Сохранить нижний блок related-карточек как единственный механизм предложенных материалов.
- Если текстовый related использовался для SEO-перелинковки, перенести нужный эффект в карточечный блок:
  - через обычные ссылки в карточках;
  - без дублирования текста в body.

## Не делать

- Не возвращать старые hardcoded цвета.
- Не добавлять второй related-блок в другом месте страницы.
- Не раздувать hero дополнительным декором без отдельной дизайн-задачи.

## Тесты

### Ручные

- Открыть `/` в light theme и убедиться, что подзаголовок hero читаем.
- Открыть `/` в dark theme и убедиться, что подзаголовок hero тоже читаем.
- Открыть 3 статьи с related-материалами и проверить:
  - внутри body нет строки `Читать также:`;
  - related-карточки внизу остались;
  - клик по карточкам работает.

### Технические

- `npm run lint`
- `npm run build`

## Критерии приёмки

- Подзаголовок hero визуально читается на production в обеих темах.
- В article body больше нет текстового related-дубля.
- Нижний блок `Читать также` карточками остаётся рабочим.
