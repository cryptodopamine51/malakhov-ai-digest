## Концепция

Editorial минимализм в духе качественных научных изданий (NYT Science, MIT Tech Review).
Дизайн строится на контрасте: белый фон, чёрная типографика, живые фотографии.
Главный визуальный ход — wireframe-иллюстрации, вписанные в заголовки секций (для AI: схемы нейросетей, процессоры, абстрактные геометрические структуры).

**Ключевые принципы:**
- Типографика первична — контент важнее декора
- Меньше скруглений — острые углы создают серьёзный, редакционный характер
- Один акцентный цвет, использовать экономно
- Читаемость на русском языке — приоритет

## Темы (светлая / тёмная)

**Светлая тема — default.** Сайт редакционный, мы хотим консистентного вида у всех новых посетителей независимо от системных настроек.

Правила инициализации (`app/layout.tsx`, inline `themeScript`):

- Если в `localStorage` лежит `theme = "dark"` — рендерим тёмную тему.
- Во всех остальных случаях (нет ключа, `theme = "light"`, или ошибка чтения) — светлая.
- `prefers-color-scheme: dark` **намеренно игнорируется**: наш дефолт перебивает системное предпочтение. Пользователь может переключиться вручную через тоггл в хедере, и его выбор сохранится.

В хедере — переключатель тем (иконка солнце/луна, `src/components/ThemeToggle.tsx`). При клике пишет `theme = "light" | "dark"` в `localStorage` и обновляет атрибут `data-theme` на `<html>`.

## Типографика

### Шрифты

| Применение | Шрифт | Вес | CSS-переменная |
|---|---|---|---|
| Заголовки H1–H2, display | **Onest** (geometric sans) | 700–900 | `--font-onest` / `font-serif` |
| Body text, UI, навигация | **Golos Text** (humanist sans) | 400–600 | `--font-golos` / `font-sans` |
| Данные, числа, метки, kicker | **IBM Plex Mono** (monospace) | 400–700 | `--font-mono` / `font-mono` |
| Pull-quote | Onest bold | 700 | — |

> **Onest** — создан для цифровых продуктов, отличная кириллица, геометрический гротеск. Используется в tech-компаниях СНГ. Заменил Playfair Display (газетный serif не соответствовал AI/tech вектору издания).
> **Golos Text** — российский шрифт, превосходная читаемость русского текста на любых размерах.
> **IBM Plex Mono** — моноширинный шрифт IBM, задаёт технический характер числам и данным в редакционных блоках.

**Правило использования mono:** все числовые значения в статблоках, kicker-подписи, метаданные источника/даты в нижних колонтитулах, метки категорий в редакционных блоках — рендерятся `font-mono`.

### Правила читаемости

- Body text `max-width: 640px` — критично для русского языка
- `letter-spacing: 0.06em` для uppercase элементов (теги, навигация)
- Заголовки карточек: `-webkit-line-clamp: 3` — обрезать после 3 строк

## Компоненты

### Article cover fallback lab

Неиндексируемая страница `/demo/image-lab` используется как визуальная лаборатория для обложек статей без хорошей картинки.
Базовая бесплатная стратегия должна выглядеть как редакционная упаковка Malakhov AI Digest: muted palette, paper frame,
grain/line layers, строгая геометрия и понятная тематическая метка. Stock-фото нельзя использовать как самостоятельную
заглушку: поверх него нужен фирменный editorial treatment. Платная AI-генерация допустима только как тонкий слой для
важных материалов и должна визуально продолжать эту систему.

### Хедер

```
[ЛОГО]    Индустрия  Исследования  Лаборатории  Россия  Код    [🔍] [☀/🌙] [☰]
```

- Фиксированный (`position: sticky top-0`)
- Фон: `rgba(255,255,255,0.95)` + `backdrop-filter: blur(8px)`
- Нижняя линия `1px solid #E5E5E5` появляется при скролле
- Лого: жирный гротеск, ~18px, буквы логотипа
- Категории: Golos Text, 12px, uppercase, `letter-spacing: 0.08em`
- Переключатель темы: иконка 20×20, между поиском и бургером
- Мобайл: лого + иконки (поиск, тема, бургер)

### Карточка статьи (стандарт)

```
[Изображение 16:9, border-radius: 4px]
[ТЕГ] [ТЕГ]                        ← outlined, 11px uppercase, border-radius: 3px
Заголовок жирным, 3 строки max
Источник [favicon]          Дата    ← #888888, 13px
```

- Hover: `transform: translateY(-2px)`, тень `0 4px 16px rgba(0,0,0,0.08)`
- Заголовок hover: цвет → акцент `#0055FF`
- Нет box-shadow в обычном состоянии

### Featured карточка (главная в секции)

- Изображение на полную ширину карточки
- Gradient overlay снизу: `linear-gradient(transparent, rgba(0,0,0,0.75))`
- Теги + заголовок + "Читать" кнопка — поверх изображения, белым текстом
- Кнопка "ЧИТАТЬ": `border: 1.5px solid #FFFFFF`, `border-radius: 4px`, белый текст, прозрачный фон

### Теги / бейджи

- Outlined: `border: 1px solid currentColor`, `border-radius: 3px`, padding `3px 8px`
- Активный / selected: чёрный фон, белый текст
- Специальный "НОВОЕ": акцентный цвет `#0055FF`, белый текст
- Горизонтальный scroll-ряд тегов: скрыть scrollbar, но оставить функциональность

### Ряд тегов разделов (TopicTabs)

`src/components/TopicTabs.tsx` — горизонтальный ряд всех разделов сайта.
Используется на главной (под hero) и на страницах разделов (под hero, перед контентом).
Источник правды по списку разделов и их порядку — массив `TOPIC_TABS` внутри компонента.

- Тег: outlined, `border: 1px solid var(--line)`, `border-radius: 3px`, padding `3px 8px`,
  11px uppercase, `letter-spacing: 0.06em`, цвет `var(--muted)`.
- Hover: рамка и текст `var(--ink)`.
- Активный (`activeHref` совпадает): чёрный фон `var(--ink)`, белый текст `var(--base)`.
  Атрибут `aria-current="page"` для скринридеров.
- На узких экранах ряд скроллится горизонтально, scrollbar скрыт. Контейнер расширяется
  на `-mx-4 px-4` чтобы первый/последний тег не прилипал к краю.

Подсветка `/categories/ai-russia` показывается на теге `/russia`, потому что Россия — отдельная
страница (`app/russia/page.tsx`), а не category-маршрут.

### Главная: блок дня (VC-style)

Под hero-блоком и `TopicTabs` главной (только на первой странице, когда есть подходящая
статья) — двухколоночный блок «пульс дня» (`src/components/PulseList.tsx` +
featured-вариант `ArticleCard` + источник данных `getHotStoryOfTheDay` / `getRecentHeadlines`):

- Десктоп (`lg+`): grid `lg:grid-cols-5`. Левая колонка `lg:col-span-2` (~40%) —
  список из 8 свежих заголовков чисто по хронологии. Правая колонка `lg:col-span-3` (~60%) —
  одна горячая статья дня большой featured-карточкой.
- Мобайл: стек, сверху список «Свежие заголовки», под ним «Главное сегодня».
- Заголовок секций (H2 28px по типографике) — `font-serif`, `text-2xl`, чёрный.
- Статья из горячей не повторяется в списке слева (исключается через `excludeIds`).

Логика выбора горячей:
1. Самая высоко оценённая статья из last 24h среди live.
2. Если её score < порога (`scoreThreshold`, по умолчанию 5) — самая свежая опубликованная.
3. При равных score сортировка вторична по `created_at desc` — это делает результат
   воспроизводимым между двумя последовательными запросами.

Если `getHotStoryOfTheDay()` ничего не нашёл (база пуста), весь блок не рендерится —
сразу идёт стандартная лента «Все новости».

### Время «N минут назад» (RelativeTime)

`src/components/RelativeTime.tsx` — клиентский компонент.

- SSR рендерит абсолютное московское время (`HH:mm` для сегодня, `D MMM HH:mm` иначе)
  чтобы избежать hydration mismatch.
- После гидрации переключается на относительный формат через `Intl.RelativeTimeFormat('ru', { numeric: 'auto' })`.
- Тикает раз в минуту через `setInterval`.
- В `<time dateTime>` и `title` всегда лежит ISO-строка — для скринридеров и tooltip.
- Используется в `PulseList` рядом с категорией.

### Cookie-баннер и согласие

`src/components/ConsentManager.tsx` (banner + modal) + `lib/consent.ts` (хранилище) +
`src/components/Analytics.tsx` (загрузчик Метрики) — реализация согласия по 152-ФЗ.

- Баннер фиксированный внизу, появляется при первом заходе или после очистки `consent_v1`
  в localStorage. Три действия: «Принять все» (primary, чёрный), «Только необходимые»
  (secondary), «Настроить» (text-link).
- Модалка «Настроить» открывается поверх баннера, перечисляет три категории
  (необходимые / аналитические / маркетинговые) с переключателями. Категория «необходимые»
  всегда включена и заблокирована.
- Решение хранится в `localStorage` под ключом `consent_v1` как объект
  `{ version, decision, categories, decidedAt }`. Версия в имени ключа — на случай смены политики.
- При сохранении ConsentManager диспатчит `window.CustomEvent('consent-changed', { detail })`.
  Аналитический загрузчик (`Analytics`) слушает событие и подгружает Метрику только при
  `categories.analytics === true`. Скрипт грузится с `next/script` `strategy="lazyOnload"`.
- Кнопка «Отозвать согласие» на `/consent` (`RevokeConsentButton.tsx`) удаляет ключ и
  перезагружает страницу — после этого баннер появляется снова.
- Юридические страницы: `/privacy-policy`, `/cookie-policy`, `/consent`. Тексты в коде
  помечены `TODO(legal)` и подлежат правке юристом перед публичным запуском.

## Страницы

### Главная `/`

1. **Hero-блок** в тонкой рамке:
   - Название издания гигантским serif (~120px)
   - Поверх — wireframe-иллюстрация AI-тематики (нейросеть, схема чипа)
   - Под ним — 2 равные карточки с изображением

2. **Ряд тегов разделов** под hero — `TopicTabs` без `activeHref`.
   Пользователь сразу видит все разделы и может переключиться без возврата на главную.

3. **Блок дня** (VC-style) — список свежих заголовков слева + featured горячая статья
   справа. Подробности — секция «Главная: блок дня (VC-style)» выше.

4. **Лента «Все новости»** одна общая, без разбивки по секциям. Пагинация вперёд/назад.

**Responsive правило для листингов:**
- mobile: 1 колонка
- tablet: 2 колонки
- wide desktop: 3 колонки
- не включать 3 колонки слишком рано на ширинах около `768–1024px`

### Страница категории `/categories/[slug]` и `/russia`

- Hero: название категории крупным (H1 48px desktop, line-height 1.1) + wireframe-иллюстрация, всё в рамке.
- Лид (1–2 предложения, что в этом разделе) — `var(--muted)`, под H1.
- **Под hero — `TopicTabs` с `activeHref` текущего раздела.** Активный тег подсвечен чёрным фоном.
- Далее: featured статья на полную ширину.
- Затем: сетка 3 колонки на широких экранах.
- Пагинация или infinite scroll.
- Боковых описательных блоков нет: всё описание раздела живёт в hero-лиде.

### Страница статьи `/categories/[category]/[slug]`

> Хлебные крошки: `Главная → <Категория> → <Источник>`. Категория — кликабельная ссылка
> на `/categories/<primary>`. Legacy `/articles/<slug>` отвечает 308-редиректом и оставлен
> только для совместимости со старыми ссылками.

**Desktop:**
- Хедер-блок в рамке: wireframe + теги + H1 + автор/дата
- Двухколоночный layout: левая 25% (источник, курсивная заметка) + правая 70% (текст)
- Лид: 20px, жирный
- Pull-quote: курсив, тонкая рамка слева `3px solid #E5E5E5`
- Изображения: в рамке `1px solid #E5E5E5`, caption под ним серым
- В конце: шеринг + ← НАЗАД / ВПЕРЁД →
- "Похожие статьи" — 4 карточки в ряд
- "Популярное в категории" — mixed layout

**Мобайл:**
- Иллюстрация/фото полной ширины сверху
- Теги, H1, автор/дата
- Единая колонка, `padding: 0 16px`

**Responsive правило для чтения:**
- на ширинах `768–1024px` статья не должна уходить в тесную двухколоночную схему;
- sidebar появляется только когда основная текстовая колонка сохраняет комфортную ширину;
- блоки `lead`, `summary`, `video`, `glossary` должны иметь заметные вертикальные паузы и не сливаться в один стек карточек.

**UX-улучшения:**
- Прогресс-бар чтения: `4px` линия под хедером, акцентный цвет
- Sticky хедер показывает заголовок статьи при скролле вниз
- Время чтения рядом с датой: "5 мин"

### Страница источников `/sources`

- Карточки источников: логотип + название + описание + количество статей + последняя дата
- При клике — фильтрованный список статей этого источника

## Иконки и иллюстрации

- Иконки: Lucide React (линейные, stroke-based, 20×20)
- Wireframe-иллюстрации для hero-блоков: SVG, чёрные линии на белом (`stroke: #111`, `fill: none`)
  - Главная: абстрактная нейронная сеть или процессор
  - Индустрия: схема бизнес-процесса
  - Исследования: молекула или граф
  - Лаборатории: лабораторная колба / формула
  - Россия: карта-контур
  - Код: терминал / bracket схема
- Фавиконки источников: загружаются через `favicon.im` или хранятся локально
