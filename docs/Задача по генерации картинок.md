# **1\. Что я рекомендую как финальную схему**

## **Роль пайплайна**

Пайплайн лучше разделить на 2 слоя:

### **Слой A — «visual director»**

Он читает статью и выдаёт:

* рубрику,  
* визуальное семейство,  
* главную метафору,  
* scene type,  
* motif tags,  
* финальный prompt для Imagen.

### **Слой B — «image renderer»**

Он:

* выбирает модель,  
* отправляет prompt в Imagen,  
* сохраняет результат,  
* пишет в БД метаданные и стоимость.

Для Imagen 4 у тебя есть понятная ценовая лестница:  
`Imagen 4 Fast` — **$0.02/image**, `Imagen 4` — **$0.04/image**, `Imagen 4 Ultra` — **$0.06/image**. Это делает очень удобным роутинг по важности статьи. ([Google Cloud](https://cloud.google.com/vertex-ai/generative-ai/pricing))

# **2\. Финальный управляющий prompt**

Это prompt для **LLM-оркестратора**, который **не рисует картинку сам**, а готовит качественный prompt для Imagen 4\.

## **SYSTEM PROMPT — Visual Director**

You are the visual director for Malakhov AI Digest, a premium AI news publication.

Your task is to convert each article into a high-quality image generation brief for Imagen 4 on Vertex AI.

Publication identity:  
Malakhov AI Digest uses a premium editorial conceptual photomontage style with strong magazine-cover character. The visual language combines layered paper collage logic, restrained modernist forms, asymmetrical editorial composition, matte paper texture, soft print grain, and clear symbolic metaphors. The publication should feel intelligent, media-grade, serious, and visually cohesive across articles.

Your job:  
1\. Read the article input.  
2\. Identify the article category.  
3\. Identify the main editorial angle.  
4\. Choose one strong visual metaphor.  
5\. Choose one scene type.  
6\. Generate one final production-ready image prompt for Imagen 4\.  
7\. Generate motif tags for repetition control.  
8\. Return only structured JSON.

Hard style rules:  
\- Keep a consistent publication identity across all covers.  
\- Use strong editorial photomontage language.  
\- Prefer layered paper collage, symbolic objects, architecture, media artifacts, public institutions, documents, maps, silhouettes, and selective interface fragments.  
\- Prefer asymmetrical editorial composition.  
\- Keep one clear visual center and one memorable metaphor.  
\- Preserve a restrained palette: muted coral, mustard, teal, dusty blue, graphite, off-white, warm gray.  
\- Aim for a premium magazine-cover result, not a product explainer.

Anti-template rules:  
\- Avoid repeated series shortcuts.  
\- Avoid overusing glowing network spheres, centered business figures, floating dashboard tiles, browser-window stacks, world-map-plus-chip cliché, and repeated shield compositions.  
\- Avoid placing explicit text labels like “AI” inside the image.  
\- Avoid overusing readable text blocks inside the image.  
\- Prefer symbolic forms over literal labeling.  
\- Rotate visual language across the series.

Scene types:  
\- HUMAN\_SYSTEM  
\- OBJECT\_SYMBOL  
\- SOCIAL\_EDITORIAL  
\- ARCHITECTURE\_TECH  
\- INTERFACE\_METAPHOR  
\- ABSTRACT\_CONCEPT

Visual family mapping:  
\- Models / launches → HUMAN\_SYSTEM or INTERFACE\_METAPHOR  
\- Regulation / law / governance → ARCHITECTURE\_TECH or OBJECT\_SYMBOL  
\- Safety / transparency / opacity → ABSTRACT\_CONCEPT or OBJECT\_SYMBOL  
\- Media / journalism / newsroom → SOCIAL\_EDITORIAL  
\- Open-source / community → SOCIAL\_EDITORIAL or OBJECT\_SYMBOL  
\- Markets / geopolitics / infrastructure → ARCHITECTURE\_TECH or OBJECT\_SYMBOL

Prompt-writing rules:  
\- Write in English.  
\- Write one concise but vivid production prompt.  
\- Make the image brief specific enough for generation quality, but clean enough to avoid clutter.  
\- Focus on one strong metaphor, 2–4 main elements, and one compositional direction.  
\- Keep the result suitable for a 16:9 news cover image.

Return JSON in this exact structure:  
{  
  "category": "",  
  "importance\_tier": "hero|standard|long\_tail",  
  "visual\_family": "",  
  "scene\_type": "",  
  "editorial\_angle": "",  
  "visual\_metaphor": "",  
  "motif\_tags": \[\],  
  "anti\_motifs": \[\],  
  "render\_prompt": ""  
}

# **5\. Финальный production prompt template для Imagen 4**

Это уже шаблон, который разработчик будет склеивать из JSON:

Create an original editorial conceptual photomontage for a premium AI news digest cover image, 16:9.

Topic: {topic}  
Core editorial angle: {editorial\_angle}  
Visual metaphor: {visual\_metaphor}

Scene direction:  
Use a {scene\_type\_humanized} composition with asymmetrical editorial balance.  
Include {main\_elements}.  
Keep one clear focal point, strong editorial hierarchy, thoughtful negative space, and a magazine-cover quality.

Art direction:  
Strong editorial photomontage, layered paper collage logic, conceptual print-era media design, matte paper texture, soft print grain, subtle halftone residue, restrained modernist geometry, premium publication finish.

Palette:  
Muted coral, mustard, teal, dusty blue, graphite, off-white, warm gray.

Series control:  
Avoid repeated series shortcuts.  
Avoid glowing network spheres, browser-window stacks, centered business-hero composition, floating dashboard tiles, world-map-plus-chip cliché, overused shield symbolism, explicit “AI” text labels, and readable text blocks inside the image.

Output qualities:  
Sophisticated, media-grade, concept-driven, serious, visually memorable, suitable for a premium AI publication.

# **8\. Что обязательно учесть в разработке**

## **1\. Кэш**

Кэшировать по:

* `article_hash`  
* `prompt_version`  
* `visual_family`  
* `render_prompt_hash`

Если текст чуть обновился, а визуальный смысл тот же, повторный рендер можно не запускать.

## **2\. Ограничение повторов**

Перед генерацией проверять последние 20–30 материалов:

* shield  
* chip  
* map  
* fragmented\_face  
* parliament  
* newsroom  
* portal  
* legal\_documents

Если мотив повторялся слишком часто, router должен сдвигать сцену.

## **3\. Один рендер по умолчанию**

Базовый режим:

* `number_of_candidates = 1`

Повторный рендер запускать только по флагам:

* `failed_quality_check`  
* `motif_conflict`  
* `editor_manual_retry`

Так ты удержишь расход под контролем.

## **4\. Текст внутри изображения**

Для прод-режима держать:

* `allow_text_in_image = false`

Imagen умеет хороший визуальный стиль, а читаемый текст в картинке обычно даёт скачки качества. Это уже практическая рекомендация из твоих тестов.

## **5\. Ручной fallback**

Если генерация дала слабый результат:

* article cover family image по рубрике,  
* либо предыдущий approved variant.

## **6\. Версионирование prompt’ов**

Поле в БД:

* `prompt_version = "v3.0"`

Иначе через месяц будет трудно понять, почему серия «поехала».

## **7\. Очередь и лимиты**

Для Imagen 4 в документации указано ограничение **30 regional online prediction requests per base model per minute per base model**. Для новостника этого обычно хватает, но worker и retry policy всё равно лучше строить через очередь. ([Google Cloud Documentation](https://docs.cloud.google.com/vertex-ai/generative-ai/docs/models/imagen/4-0-generate))

# **10\. Рекомендованный workflow**

## **Этап 1**

Статья опубликована → worker отправляет текст в `visual-director`.

## **Этап 2**

`visual-director` возвращает JSON.

## **Этап 3**

`router` выбирает модель:

* fast / standard / ultra

## **Этап 4**

`imagen-client` делает generation request.

## **Этап 5**

`validator` проверяет:

* пустых артефактов нет,  
* повторяемость мотива в допустимых пределах,  
* в prompt не попали запрещённые мотивы,  
* результат привязан к нужной рубрике.

## **Этап 6**

Сохранение в storage \+ DB.

* `IMAGEN_FAST_MODEL=imagen-4.0-fast-generate-001`  
* `IMAGE_PROMPT_VERSION=v3.0`  
* `IMAGE_ALLOW_TEXT=false`  
* `IMAGE_DEFAULT_ASPECT=16:9`

### **Обязательные бизнес-правила**

* одна генерация на материал по умолчанию;  
* rerender только по причине;  
* motif-history обязателен;  
* category families фиксированы;  
* prompt versioning обязателен.
