# Спека: локальная генерация изображений через ComfyUI на сервере

## Контекст

Сейчас для визуалов есть два неудобных полюса:

- SVG и кодовые схемы слишком слабые для обложек и hero-визуалов;
- внешние image API дают зависимость от провайдера, лимитов, биллинга и сети.

Нужна серверная локальная генерация без внешних image API, чтобы пайплайн мог стабильно работать независимо от ноутбука и выдавать визуал уровня editorial collage / photomontage.

## Цель

Поднять на сервере постоянный сервис генерации на базе `ComfyUI`, доустановить нужные модели, собрать workflow под стиль `editorial collage / photomontage` и провести тест на 10 изображений, чтобы оценить качество и пригодность для прод-использования.

## Что должно получиться

- На сервере работает отдельный сервис `ComfyUI`, не завязанный на локальный компьютер.
- Есть воспроизводимый workflow под формат `16:9` для новостных обложек.
- Есть набор установленных моделей и LoRA под editorial / collage / photomontage.
- Есть 10 тестовых изображений по заранее заданному пакету промптов.
- Есть понятные критерии оценки качества и решение, годится ли стек в прод.

## Ограничения

- Без внешних image API.
- Без обязательной зависимости от SaaS-оркестратора.
- Запуск и генерация должны работать после перезагрузки сервера.
- Конфигурация должна быть воспроизводимой: через `docker compose` или через документированный systemd/venv-setup.
- Генерация должна быть доступна только нам: через VPN, basic auth или reverse proxy allowlist.

## Рекомендуемая архитектура

### Базовый вариант

- `ComfyUI` как отдельный сервис на сервере.
- GPU-нодa с постоянным хранилищем моделей.
- Reverse proxy через `Caddy` или `Nginx`.
- Отдельный volume для:
  - `models/checkpoints`
  - `models/loras`
  - `models/controlnet`
  - `input`
  - `output`
  - `user/default/workflows`

### Предпочтительный способ запуска

`docker compose`, потому что так проще:

- переживать перезапуски;
- фиксировать версию окружения;
- переносить конфигурацию на другой сервер;
- не ловить дрейф зависимостей Python/CUDA.

## Требования к серверу

### Минимум для адекватного теста

- Ubuntu 22.04/24.04
- NVIDIA GPU
- VRAM от 16 GB
- RAM от 32 GB
- SSD от 150 GB свободного места
- установленный `nvidia-container-toolkit`, если идём через Docker

### Предпочтительно

- VRAM 24 GB+
- RAM 64 GB
- отдельный диск/раздел под модели и outputs

Если VRAM меньше 16 GB, качество и скорость сильно упрутся в компромиссы, и тест 10 картинок будет менее показательным.

## Набор моделей для первой итерации

Нужен не один чекпоинт, а внятный стек.

### Основной трек

- `FLUX.1-dev` или совместимая локальная сборка/квантованная версия, если сервер тянет по VRAM
- `FLUX.1-schnell` для быстрых черновиков и prompt-итераций

### Fallback-трек

- один сильный `SDXL`-чекпоинт под фотореалистичный editorial
- один `SDXL`-чекпоинт под stylized collage / print texture

### Дополнительно

- 2-4 LoRA для:
  - editorial magazine look
  - paper collage / cutout layering
  - print grain / matte texture
  - photomontage composition
- апскейлер для финального рендера
- при необходимости `ControlNet` или аналог для композиционного контроля

## Что именно нужно настроить в ComfyUI

### Workflow v1

Собрать один рабочий workflow под `editorial collage / photomontage`:

1. Вход:
   - positive prompt
   - negative prompt
   - seed
   - aspect ratio `16:9`
   - quality preset `draft | standard | hero`
2. Генерация композиции:
   - checkpoint/model loader
   - text encode
   - sampler
3. Стайлинг:
   - LoRA stack для editorial + collage
   - print grain / matte finish
4. Улучшение:
   - hi-res pass или latent upscale
   - optional detail pass для лиц/архитектуры, если это не ломает стиль
5. Выход:
   - PNG/WebP
   - сохранение prompt metadata
   - predictable file naming

### Workflow v2

Собрать вариант с более жёстким контролем композиции:

- input reference layout или simple mask
- optional image-to-image / style reference
- отдельный путь для hero-визуалов

## Формат генерации

### Черновики

- `1344x768` или близко
- быстрый preset

### Финал

- `1792x1024` или `1920x1080`
- upscaler pass
- один и тот же seed должен воспроизводить результат достаточно близко

## Пакет теста на 10 изображений

Нужен не “10 рандомных картинок”, а контролируемый benchmark.

### Набор сюжетов

Сделать 10 изображений по 10 разным сюжетам:

1. Регуляция AI: институции, документы, технологическая инфраструктура.
2. Запуск новой модели: человек + интерфейс + символическая метафора.
3. Open source: сообщество, код, сборка, мастерская.
4. Чипы и инфраструктура: архитектура, дата-центр, supply chain.
5. Медиа и журналистика: newsroom, коллаж из документов, экранов и вырезок.
6. Безопасность и transparency: стекло, слои, маски, частичная видимость.
7. Роботизация труда: человек-система, производственная среда.
8. Образование и AI: аудитория, доска, интерфейсные артефакты.
9. Геополитика AI: карты, границы, институциональная графика, без штампов.
10. Российский AI-контекст: локальная инфраструктура, индустрия, редакционный тон без клюквы.

### Для каждого сюжета сохранить

- исходный prompt
- negative prompt
- seed
- модель
- LoRA stack
- sampler settings
- время генерации
- итоговый файл

## Критерии качества

Картинка считается удачной только если одновременно выполняются условия:

- выглядит как editorial visual, а не как типичный AI-slop;
- есть один сильный визуальный центр;
- нет клише вроде glowing orb, chip-on-globe, flying dashboard tiles;
- читается как серьёзный media-grade photomontage;
- хорошо держит `16:9` и кроп под hero;
- нет явных анатомических развалов, мусорных рук, битой перспективы, каши из объектов;
- нет слишком буквального “AI” текста внутри изображения;
- стиль серии узнаваем, но кадры не дублируют друг друга.

## Acceptance Criteria

- `ComfyUI` поднимается на сервере автоматически после рестарта.
- Все пути к моделям и output-директориям вынесены в конфиг или volume mapping.
- Workflow экспортирован и хранится в репозитории или в отдельной infra-папке.
- 10 тестовых изображений сгенерированы и сохранены.
- По каждому тесту есть таблица параметров и короткая оценка.
- Минимум 4 из 10 изображений выглядят пригодно для редакционного использования без ручной перерисовки.
- Минимум 2 из 10 изображений можно ставить в hero после лёгкой пост-обработки.

## Что сделать по шагам

### Этап 1. Infra

- Поднять `ComfyUI` на сервере.
- Настроить автозапуск.
- Настроить storage для моделей и outputs.
- Закрыть публичный доступ, оставить только наш контур доступа.

### Этап 2. Models

- Скачать и разложить checkpoint-модели.
- Скачать и разложить LoRA.
- Проверить совместимость версий, нод и VRAM-потребление.

### Этап 3. Workflow

- Собрать workflow `editorial_collage_v1.json`.
- Собрать workflow `editorial_collage_v2.json`.
- Добавить presets `draft`, `standard`, `hero`.

### Этап 4. Prompting

- Зафиксировать style prompt base.
- Зафиксировать negative prompt base.
- Собрать prompt-pack на 10 сюжетов.

### Этап 5. Тест

- Сгенерировать 10 изображений.
- Сохранить все параметры генерации.
- Отобрать лучшие 3-5 результатов.
- Коротко описать, где стек сильный, а где разваливается.

### Этап 6. Интеграция в пайплайн

Сразу в первую итерацию полноценную автогенерацию в прод не вшивать. Сначала нужен полуавтоматический режим:

- локальный prompt JSON или ручной prompt input;
- запуск генерации по команде;
- сохранение результата в понятную папку;
- только после теста решать, подключать ли это к editorial pipeline.

## Риски

- На слабом GPU `FLUX` может оказаться слишком медленным для реального прод-ритма.
- Без нормального LoRA-набора картинки легко съедут в generic AI-art.
- Без negative prompt discipline серия быстро скатится в повторяющиеся штампы.
- Если сразу автоматизировать генерацию в пайплайн, можно получить много мусора. Поэтому сначала только controlled benchmark.

## Definition of Done

- Серверная локальная генерация поднята и доступна.
- Workflow под editorial collage / photomontage сохранён и воспроизводим.
- Сформирован пакет из 10 изображений.
- Есть короткий отчёт по качеству и решение:
  - либо стек годится для следующей интеграции;
  - либо меняем модели / LoRA / workflow до второго раунда.

## Короткая формулировка задачи для трекера

Поднять на сервере локальную генерацию через `ComfyUI` без внешних image API, доустановить модели и LoRA под стиль `editorial collage / photomontage`, собрать воспроизводимый workflow `16:9`, провести benchmark на 10 тестовых сюжетах и подготовить отчёт по качеству с решением о дальнейшей интеграции в контент-пайплайн.
