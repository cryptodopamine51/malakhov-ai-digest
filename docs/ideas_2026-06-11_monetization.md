# Монетизация сайта — разбор и план (2026-06-11)

> Рабочий документ (не канонический). Исследование по запросу владельца:
> «как монетизировать сайт, что писать и куда лить трафик, от меня минимум запар».
> Данные: Метрика API (живая, 2026-05-12 → 2026-06-10), выгрузка Вебмастера за год,
> проверка партнёрок (2026-06-11).

## Исходное положение

- Трафик: **~360 визитов/мес** (органика 217, прямые 119, соцсети 10). Органика растёт
  (месяц назад было 270 всего).
- Органика рассыпана по новостному лонгтейлу (топ-страница — 16 визитов/мес). Гайды
  органики пока не получают.
- Единственный устойчивый поисковый актив: статья «Как оплатить Cursor из России» —
  4 запроса с суммарным спросом ~1000–1300 показов/мес весь год, позиция 9–11.
- Монетизации нет вообще: ни партнёрок, ни рекламы; есть воронка на `/services`
  (консультации), но цели в Метрике не настроены — конверсия не измеряется.

## Пути монетизации (по приоритету при нашем масштабе)

### 1. Услуги владельца — главные деньги сейчас и на год вперёд

При 12 визитах/день любая реклама даёт копейки, а **один клиент на консультацию/внедрение
окупает годы рекламного дохода**. Весь коммерческий контент должен в первую очередь
кормить эту воронку.

Что делаю сам:
- Настраиваю цели в Метрике через Management API (проверено: токен из
  `malakhov-ai-keys.env` имеет права; автоцели уже собираются) — клики по CTA
  консультации, переход в личный TG, визит `/services`.
- Кластер C (см. ниже): гайды под коммерческие запросы «внедрение ИИ в бизнес»,
  «аудит процессов для ИИ» с CTA на консультацию.

### 3. Контент-план: что писать и куда лить трафик

- **Кластер A «Доступ к ИИ из России»** (монетизация: GGSel, GetPayAll, VseGPT, GPTunnel):
  - «Как оплатить ChatGPT Plus из России 2026», то же про Claude Pro, Midjourney,

  - «Сервисы оплаты зарубежных подписок: сравнение» — каждая строка = партнёрская ссылка;
  - «BotHub vs VseGPT vs GPTunnel: доступ к топовым моделям из России».
  - Валидация спроса — Вордстат (паттерн подтверждён Cursor-статьёй: ~1000+ показов/мес).
- **Кластер B «Локальные LLM и железо»** (монетизация: хостинги/GPU-облака):
  - «Аренда GPU-сервера для LLM в России», «Как запустить Gemma/Qwen дома», «Бюджетная
    сборка под локальный ИИ». У нас уже есть органика на эту тему.
- **Кластер C «ИИ для бизнеса»** (монетизация: услуги владельца):
  - существующая серия гайдов + 2–3 страницы под коммерческий интент
    («внедрение ИИ под ключ», «сколько стоит ИИ-аудит») с CTA на консультацию.

### 4. Гигиена (обязательная)

- Партнёрские ссылки: `rel="sponsored"`, ставить только где сервис упомянут по делу.
  Новости линкофермой не делать — убьёт траст и SEO.
- Маркировка рекламы (ФЗ «О рекламе»): через Admitad erid проставляется автоматически;
  прямые реф-ссылки (VseGPT, gglead) — изучить до массовой расстановки, минимум — видимая
  пометка «партнёрская ссылка». Риск штрафов реальный, нужна аккуратность.
- Дисклеймер на money-страницах: «материал содержит партнёрские ссылки».

## Вордстат: статус доступа (2026-06-11)

Полученный OAuth-токен (`WORDSTAT_WORDSTAT_OAUTH_TOKEN`) — под **старый API**
`api.wordstat.yandex.net`, который **мёртв** (404, сертификат хоста уже отозван под
wordstat.yandex.ru). Вордстат переехал в **Yandex Cloud Search API v2**:
`https://searchapi.api.cloud.yandex.net/v2/wordstat/*`, авторизация — `Api-Key` сервисного
аккаунта Yandex Cloud (или IAM-токен) + `folderId`. Использование бесплатное.
Обмен текущего токена на IAM проверен — отказ: нет scope `cloud:auth`.

Минимальный путь для владельца (один клик):
1. Открыть `https://oauth.yandex.ru/authorize?response_type=token&client_id=1a6990aa636648e9b2ef855fa7bec2fb`
   (официальное приложение Yandex Cloud) → авторизоваться → скопировать токен.
2. Заменить значение `WORDSTAT_WORDSTAT_OAUTH_TOKEN` в `malakhov-ai-keys.env`.
3. Дальше сам: обмен на IAM-токен, поиск folderId через Resource Manager API,
   `scripts/wordstat-research.ts`, прогон тем кластеров A–C по частотности.
Требование: у аккаунта должно существовать облако в Yandex Cloud (создаётся автоматически
при первом входе в консоль console.yandex.cloud).

## Что нужно от владельца (всё остальное делаю сам)

   дальше ссылки расставляю я.
