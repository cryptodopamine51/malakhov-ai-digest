### B2. Wordstat-скрипт + прогон денежных тем

> ✅ **Прогон выполнен 2026-06-11 напрямую через API** — результаты и шорт-лист гайдов:
> `docs/wordstat_2026-06-11.md`. Скрипт ниже остаётся как задача для повторных прогонов
> (валидация новых тем), приоритет понижен. В скрипт заложить ретрай на разовые 403
> (наблюдался флап).

Новый `scripts/wordstat-research.ts` (read-only, без записи в БД):
- Endpoint: `POST https://searchapi.api.cloud.yandex.net/v2/wordstat/topRequests`
- Заголовки: `Authorization: Api-Key ${process.env.WORDSTAT_API_KEY}`,
  `Content-Type: application/json`
- Тело: `{"phrase": "<фраза>", "numPhrases": 50, "folderId": "b1g9m3kj0uamkcfoqmb1"}`
  (numPhrases обязателен, 1–2000; операторы `!слово`/`+слово`/`[слово]` НЕ поддерживаются)
- Ответ: `{ topRequests: [{phrase, count}], totalCount }` (+ блок похожих запросов,
  посмотреть фактический ответ). Есть также `/v2/wordstat/dynamics` (period: у MONTHLY
  toDate должен быть последним днём месяца) — для сезонности, опционально.
- Rate limit ~5–10 rps — пауза 300–500 мс между фразами.
- CLI: `npx tsx scripts/wordstat-research.ts --phrases-file=<txt со списком фраз>` или
  `--phrase="..."`. Вывод: markdown-таблица (фраза, totalCount, топ-10 вложенных с count)
  в stdout и в `docs/wordstat_<дата>.md`.
- Env-загрузка: ключ читать из `malakhov-ai-keys.env` тем же способом, каким существующие
  скрипты читают `.env.local` (посмотреть `scripts/audit-digest-selection.ts` или
  аналогичный; если читают только `.env.local` — добавить fallback на `malakhov-ai-keys.env`).
