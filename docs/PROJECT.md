# Project Overview

## Что это

Malakhov AI Digest — русскоязычное AI-медиа, которое собирает англоязычные и русскоязычные источники, отбирает важные материалы, превращает их в редакторский русскоязычный текст и публикует на сайте и в Telegram.

Проект не должен выглядеть как бездушный RSS-агрегатор. Цель — ощущение живого, отобранного медиа.

Production-редактор новостного контура — DeepSeek V4 Flash. Claude не является fallback-зависимостью:
строгая локальная валидация и bounded retry применяются одинаково ко всем категориям и risk flags.

Публичный news-контур живёт на `https://news.malakhovai.ru`. Основной домен
`https://malakhovai.ru` зарезервирован под отдельный лендинг/портфолио и не является canonical
для новостных материалов.

## Цели по приоритету

1. Интересный для чтения продукт.
2. SEO-трафик по русскоязычным AI-запросам.
3. Telegram как возвратный канал и ежедневная дистрибуция.
4. Долгосрочно — сильное тематическое медиа вокруг ИИ.

## Для кого

- Люди, которые следят за AI-индустрией, но не хотят читать десятки англоязычных источников.
- Разработчики, предприниматели, продуктовые специалисты, инвесторы.
- Пользователи, которым нужен быстрый, но качественный русскоязычный контекст по AI-новостям.

## Основные поверхности продукта

|---|---|
| Главная `/` | первое касание, «пульс дня» (свежие заголовки + горячая статья) и общая лента |
| Evergreen-гайды `/guides` и `/guides/[slug]` | статичные SEO-опорные материалы вне новостного Supabase-пайплайна |
| Услуги `/services` | индексируемая коммерческая surface под запросы «ИИ-консалтинг», «ИИ-аудит», «внедрение ИИ под ключ», «разработка ИИ-решений». Оффер — консультация; ведёт на `malakhovai.ru/contacts` (с UTM) и личный TG. `ProfessionalService` + visible FAQ/FAQPage JSON-LD. Точка приземления для CTA из статей и гайдов |
| Страница статьи `/categories/[category]/[slug]` | основная SEO и editorial surface |
| Category pages `/categories/[category]` и `/russia` | тематические SEO-кластеры: свежая лента + отдельный блок «Самое интересное» |
| Legacy `/articles/[slug]` и `/topics/[topic]` | только 308-редирект на канонический URL (волна 2.2) |
| Источники `/sources` и `/sources/[source]` | доверие, навигация по origins |
| Архив `/archive/[date]` | доступ к историческим подборкам |
| `/about` | E-E-A-T surface: что за издание, редакционная политика, технология, контакты. `AboutPage` JSON-LD |
| `/search` | site-search по `ru_title/lead/teaser/body` для navigational запросов; `noindex, follow`. Завязан на WebSite SearchAction в JSON-LD |
| `/llms.txt`, `/llms-full.txt` | LLM-консьюминг: первая — короткая карта сайта, вторая — full markdown dump (топ 100 статей + все гайды), ISR 1ч, лимит 5MB |
| `/news-sitemap.xml`, `/rss.xml`, `/sitemap.xml` | дистрибуция в новостные агрегаторы (Google News/Publisher Center) и читалки; тех-готовность ✅, подача — зона владельца (см. `docs/OPERATIONS.md` → «Новостные агрегаторы») |
| `/privacy-policy`, `/cookie-policy`, `/consent` | юридический контур по 152-ФЗ: политика ПД, политика cookies, согласие на обработку персональных данных |

| Telegram channel posts | возврат аудитории и daily delivery: 5 отдельных постов с обложкой и ссылкой на сайт в течение дня |

## Домены и SEO-границы

- `news.malakhovai.ru` — единственный canonical-домен для новостного сайта, статей, разделов,
  RSS, `llms.txt`, `robots.txt` и sitemap.
- `malakhovai.ru` — отдельный лендинг; ссылки с него на новости должны быть абсолютными
  `https://news.malakhovai.ru/...`.
- На news-сайте не использовать `malakhovai.ru` в canonical, `og:url`, sitemap или RSS links.
- `lib/site.ts` содержит жёсткий production `SITE_URL = https://news.malakhovai.ru`; SEO-файлы
  не берут домен из env, чтобы preview/staging env не мог случайно заменить canonical.
- Site identity ассеты лежат в `public/`: favicon/app icons, `brand-mark-light.png`,
  `brand-mark-dark.png` и `og-default.png`. Metadata в `app/layout.tsx` должна ссылаться
  на эти файлы напрямую, а JSON-LD Organization logo должен использовать
  `SITE_LOGO_URL = https://news.malakhovai.ru/icon-512x512.png`.

## Что обязательно для продукта

- Статьи должны быть редакторски нормализованы на русском, а не просто переведены.
- На странице статьи должен быть полезный контекст: лид, основное тело, media, related links, glossary при наличии.
- Evergreen-гайды должны жить на чистых URL `/guides/<slug>`, иметь собственные metadata,
  JSON-LD и оптимизированные локальные изображения, не завися от live-статуса в Supabase.
  На `/`, `/guides` и `/guides/[slug]` local guide images отдаются native `srcset` с
  `-480/-768` variants, чтобы мобильные/VPN пользователи не скачивали 1200px canonical без нужды.
- Публичные URL должны быть чистыми и стабильными.
- Контент должен сохранять связь с оригинальным источником и корректно его атрибутировать.
- Дефолтная визуальная подача — светлая тема. Системное `prefers-color-scheme` не перебивает дефолт; пользователь переключает тему вручную, выбор сохраняется (`docs/DESIGN.md`).
- Полный список разделов виден сразу под hero на главной и на каждой странице раздела (компонент `TopicTabs`); навигация между разделами не требует возврата на главную (`docs/DESIGN.md`).
- В конце страницы статьи после рекомендаций есть компактный ряд «Продолжить по разделам»:
  текущий primary section стоит первым, остальные идут в порядке основной навигации.
- Авторство и воронка в услуги (единый компонент `AuthorCard`, `src/components/AuthorCard.tsx`):
  - На **гайдах** — видимый байлайн (фото + имя + «проверено <дата>») в шапке и `AuthorCard`
    в конце. Гайды — экспертный контент, поэтому личное авторство Ивана Малахова видно.
  - На **новостях** — строка «Подготовлено редакцией Malakhov AI» под заголовком (новости
    генерит пайплайн, личное авторство на них не ставим) + `AuthorCard` рядом с `TelegramCTA`.
  - `AuthorCard` даёт два действия: **личный Telegram** `@iddopamine` (бренд/продажи) и
    **консультация**. Вместе с `TelegramCTA` (канал-дайджест `@malakhovaidigest`) на странице
    получается три ясных действия. Канал и личный TG — разные кнопки, не путать.
  - На гайдах/новостях кнопка консультации ведёт на `/services`; на самой `/services` —
    напрямую на `malakhovai.ru/contacts` (`AuthorCard consultationHref="contacts"`).
- Зона CTA в конце новости (порядок): мост «Разобраться глубже» на тематический гайд
  (`lib/guide-bridge.ts`, по `primary_category`; для категорий без подходящего гайда —
  пропускается) → акцентный блок «Внедряю ИИ в бизнес — обсудим задачу» → `/services` (с UTM)
  → `TelegramCTA` (канал) → `AuthorCard` (личный TG + автор). Блок related переименован в
  «Читать дальше» и усилен визуально — цель поднять глубину просмотра.

## Что не является целью

- Форум или пользовательский UGC.
- Ручное CMS-редактирование как основной сценарий.
- Поддержка legacy Python/FastAPI контура.
