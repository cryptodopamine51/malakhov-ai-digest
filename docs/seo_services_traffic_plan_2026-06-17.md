## Что не делать

- Не делать отдельную страницу только под `внедрение ии под ключ`: 85 показов/мес. Этот интент должен жить на `/services` и в FAQ.
- Не делать отдельный гайд `RAG для бизнеса`: 16 показов/мес. Для SEO лучше говорить `база знаний с ИИ`, а RAG объяснять внутри.
- Не оптимизировать P0-документы под `бесплатно`: там будет трафик, но не заявки.
- Не плодить дубли вокруг `ии для бизнеса` и `нейросети для бизнеса`: уже есть опубликованный кластер, лучше обновлять и перелинковывать.
- Не писать промышленный гайд как обзор трендов. Нужна статья для человека с процессом, данными, браком, простоем оборудования и KPI.

## Перелинковка в услуги

Каждая P0-статья должна вести в `/services` тремя путями:

1. Верхний/средний inline CTA после блока «где ИИ окупается».
2. AuthorCard внизу с консультацией.
3. Related/ctaCards на `skolko-stoit-vnedrenie-ii-v-kompaniyu`, `kak-vybrat-pervyj-ii-proekt-v-biznese`, `/services`.

На `/services` нужно добавить блок «Популярные разборы»:

- ИИ для производства
- ИИ для документов
- Как создать ИИ-агента
- ИИ для продаж
- Сколько стоит внедрение ИИ

## Метрики

Минимальный набор целей в Метрике:

- визит `/services`;

- клик в личный Telegram;

Оценка по контенту: через 4-6 недель смотреть не только клики, но и показы/позиции по хвостам. Для новых P0-статей нормальная первая цель — попасть в показы по 20-50 хвостам, затем дорабатывать title/H2 по реальным запросам Вебмастера.

## Как повторить Wordstat-прогон

```bash
npx tsx scripts/wordstat-research.ts \
  --phrases-file=docs/wordstat_service_phrases_2026-06-17.txt \
  --num=30 \
  --out=docs/wordstat_services_YYYY-MM-DD.md \
  --raw-out=/tmp/wordstat_services_YYYY-MM-DD.json
```

Скрипт берёт `WORDSTAT_API_KEY` из `malakhov-ai-keys.env`, сырые ответы в репозиторий не кладёт.
