**Автор аудита:** топ-редактор/маркетолог

## 0. Коротко: что не так и что делать

**Диагноз в одном абзаце.**
Сейчас проект — это RSS-агрегатор с машинным переводом и бесполезной врезкой «Почему это важно». На сайте читателю выдаётся сырой DeepL-перевод первых 2000 символов оригинальной статьи, а в Telegram — тот же машинный вывод, завёрнутый в пафосную обёртку «💡 Почему это важно». Это не медиа. Это фид. Люди такое не читают, не пересылают, не подписываются и не запоминают.

**Главная ошибка стратегии.** Вы пытаетесь объяснять новости вместо того, чтобы их **продавать**. «Почему это важно» — редакторский инструмент для длинной аналитики в Bloomberg или The Information. В русскоязычной новостной ленте про ИИ нужно не «почему», а **хук**: что случилось + одна конкретная деталь, из-за которой хочется кликнуть. Всё.

**Что меняем по приоритету.**
1. Убить «Почему это важно» как концепт. Заменить на **два поля: `lead` (1–2 предложения-хука)** и `summary` (3–5 тезисов для тех, кто открыл статью).
2. Переписать промпт Claude с нуля. Писать на тоне «Коммерсантъ / vc.ru», а не «ChatGPT на русском».
3. Ввести редакторский порог публикации (см. фазу 2 в `docs/newsroom_product_plan.md`) — слабые материалы не должны появляться на сайте вообще.
4. Переупаковать Telegram: формат «заголовок + 1–2 строки» вместо текущей 3-блочной конструкции.
5. Добавить SEO-контур: JSON-LD NewsArticle, грамотные meta-description’ы, топик-лендинги, внутреннюю перелинковку.
6. Включить воронку: сайт → Telegram → продукты.

## 1. Как это выглядит сейчас глазами читателя

### 1.1 Сайт (`news.malakhovai.ru`)

Пользователь заходит на главную. Видит:
- **Заголовок страницы**: «Последние новости». Плоско, не цепляет, SEO 0 баллов. Нет ни даты выпуска, ни подзаголовка, ни brand-promise.
- **Featured-карточка**: обложка + машинный перевод заголовка + `why_it_matters` (если сгенерирован). Из-за текущего промпта этот `why_it_matters` звучит как «Эта новость важна, потому что она показывает, что ИИ продолжает развиваться и влиять на индустрию» — пустая абстракция.
- **Сетка 3×N карточек**: те же проблемы — машинные заголовки и «почему важно».
- **Клик на карточку**. Открывается страница статьи: хлебные крошки → картинка → заголовок → `SourceBadge + time + topics` → **врезка «💡 Почему это важно»** с машинным текстом → **основной текст**, который является **первыми 2000 символами оригинала после DeepL**. Это:
  - начинается с корпоративного лида источника («В понедельник компания OpenAI объявила...»),
  - содержит англоязычные хвосты типа «в своём официальном заявлении»,
  - обрывается на середине мысли (потому что 2000 символов),
  - не имеет подзаголовков, выделений, цитат, цифр.
- **Футер**: «Читать оригинал →». Пользователь уходит на английский источник.

**Итог для читателя:** «Это кривой перевод новости, которую я мог прочитать в оригинале. Зачем мне этот сайт?»

### 1.2 Telegram

Формат сообщения в `bot/daily-digest.ts:113`:

```
🤖 AI Дайджест | Malakhov AI — 16 апреля
Лучшие новости за день по версии редакции

**1. Заголовок после DeepL**
💡 Эта новость важна, потому что она показывает, что...
Читать на сайте →

**2. ...**

——————————————
📰 Все новости: news.malakhovai.ru
📢 @malakhovaidigest
```

**Проблемы по маркетингу.**
- Заголовок дайджеста «Лучшие новости за день по версии редакции» — пустой promise без доказательств.
- Эмодзи 💡 перед каждым пунктом — визуальный шум. Читатель видит **одинаковую колбасу** из 5 блоков.
- Строка «Почему это важно» — это **метатекст о новости**, а не **информация из новости**. Человек должен кликнуть на сайт, чтобы узнать, что вообще произошло. Это убивает CTR: читатель либо кликает, не понимая зачем, либо не кликает вообще.
- «Читать на сайте →» — финальный CTA слабый. Он не обещает ничего сверх того, что уже есть в сообщении.
- Нет графики. Telegram поддерживает превью ссылок — сейчас оно **отключено** (`disable_web_page_preview: true`). Это убивает CTR в 3–5 раз.
- Нет хэштегов → нет поиска внутри TG по темам.
- Нет разбивки по секциям (Россия / Мир / Лаборатории / Разработчикам) — читатель, которому интересен только Claude, видит то же, что и читатель, которому интересен только Яндекс GPT.

**Итог для читателя:** «Очередной канал-агрегатор. Похоже на 50 других. Отпишусь.»

## 2. Почему это критично для ваших целей

Вы сформулировали три стратегические задачи:

1. **Ежедневная аудитория медиа.** Люди должны ждать выпуск, читать его и рекомендовать.
2. **SEO-трафик.** Органика через Google/Яндекс на русскоязычную AI-тему.
3. **Монетизация.** Продажа ваших продуктов через медиа (сайт + Telegram).

Текущая архитектура **не решает ни одну** из них.

| Цель | Что нужно | Что есть сейчас | Разрыв |
|---|---|---|---|
| Ежедневная аудитория | Уникальный голос, редакционная позиция, узнаваемый формат | Машинный перевод + абстрактная врезка | Нет голоса, нет формата |
| SEO-трафик | Уникальный текст, грамотная структура, JSON-LD, внутренняя перелинковка, долгие страницы | Куски оригинала после DeepL; краткий `<head>`; нет перелинковки; страницы 1500 знаков | Google видит дубликат источника → не ранжирует |
| Монетизация | Воронка: TG → сайт → лид-магнит / продукт | Нет CTA на продукты, нет email-capture, нет landing-страниц под офферы | Воронки просто нет |

**Оценка.** При текущем качестве контента канал не наберёт 3000 подписчиков за полгода даже с платной рекламой. SEO-трафика не будет до тех пор, пока на сайте сырые куски переводов. Продукты продавать не из чего.

## 3. Проблемы по категориям (полный список)

### 3.1 Качество текста и редактура — **P0 (блокирует всё)**

- **P0-1.** `ru_text` — это первые 2000 символов оригинала после DeepL, без редакторской обработки (`pipeline/fetcher.ts:15`, `pipeline/enricher.ts:116`). Читается как машинный перевод.
- **P0-2.** Промпт Claude генерирует абстрактные «выводы о важности» (`pipeline/claude.ts:44–52`). Формулировка «почему эта новость важна для людей, следящих за AI-индустрией?» провоцирует модель выдавать мета-рассуждения.
- **P0-3.** Нет «лида» — первой строки-хука с фактом. Пользователь начинает чтение с чужого корпоративного ввода.
- **P0-4.** Нет стиль-гайда. Нет тона голоса. Разные статьи звучат по-разному (DeepL переводит по-своему, Claude — по-своему).
- **P0-5.** Нет проверки на «слабый материал» перед публикацией. Любая статья с `score >= 2` уходит на сайт.
- **P0-6.** Нет дедупликации по смыслу (кластеризации) — в ТГ может уйти 2 новости об одном событии с разных источников.

### 3.2 Telegram-дайджест — **P0**

- **P0-7.** Формат «заголовок + `💡 Почему это важно` + ссылка» не работает. Нужно: «хук-заголовок + 1–2 строки с конкретикой + ссылка».
- **P0-8.** `disable_web_page_preview: true` — отключены превью. Превью с обложкой поднимает CTR.
- **P0-9.** Нет графического топпера дайджеста (баннер с датой и темой дня).
- **P0-10.** «Лучшие новости за день по версии редакции» — пустой слоган. Нужно про конкретику: «5 из 47 новостей — коротко, по делу».
- **P0-11.** Нет сегментации: один поток для всех подписчиков. Нет формата «только про Россию», «только про лаборатории».
- **P0-12.** Нет breaking news — критичные новости ждут 24 часа до 09:00 МСК.
- **P0-13.** Нет ежедневной рубрики/ритуала (например: «Заголовок дня», «Цифра дня», «Разбор недели»). Читатель не получает от канала ничего, кроме ленты.

### 3.3 Сайт — UX и дизайн — **P1**

- **P1-1.** Главная = просто «Последние новости». Нет герой-блока выпуска, нет категорий дня, нет боковой колонки с подборками.
- **P1-2.** Страница статьи не имеет: подзаголовков, выделений цитат, связанных материалов, кнопок шеринга, комментариев/реакций.
- **P1-3.** Тёмная тема жёстко задана (`app/globals.css`). Нет переключателя, нет аналитики предпочтений.
- **P1-4.** Карточки все одинакового веса — визуально невозможно понять, что «главное» на сегодня.
- **P1-5.** Мобильный опыт не проверен отдельно — судя по `ArticleCard.tsx`, sm:grid-cols-2 пропускает промежуточный брейкпоинт для планшетов.
- **P1-6.** Нет страницы «О проекте» / «Редакция» — это убивает SEO-доверие (E-E-A-T).
- **P1-7.** Нет страницы «Архив выпусков» (`/digest`), хоть она и есть в плане.

### 3.4 SEO и структура контента — **P1**

- **P1-8.** `<meta description>` = `why_it_matters ?? ru_text.slice(0, 160)` (`app/articles/[slug]/page.tsx:27`). В выдаче Google это рендерится плохо: обрывается на середине слова, содержит машинные связки.
- **P1-9.** Нет JSON-LD `NewsArticle` schema. Без него статьи не попадают в Google News и Top Stories.
- **P1-10.** Нет канонической ссылки на оригинал (может повлечь duplicate content при индексации переводов).
- **P1-11.** Slug = транслит заголовка + суффикс UUID. Лучше: `YYYY-MM-DD-ключевое-слово`. Для SEO — важна дата.
- **P1-12.** Нет топик-лендингов с длинным SEO-текстом. `/topics/[topic]` — это просто список статей. Без H1+H2+интро на 300+ слов страница не ранжируется по запросу «новости про ИИ».
- **P1-13.** Нет sitemap-приоритетов (свежее = выше).
- **P1-14.** Нет внутренней перелинковки «связанные материалы», «читать по теме». Это ломает распределение link juice.
- **P1-15.** Нет русскоязычного RSS для сайта → сторонним агрегаторам нечего забирать.
- **P1-16.** Нет Open Graph картинок, сгенерированных динамически для соцшеринга. `/og-default.png` — статичная заглушка.

### 3.5 Воронка и монетизация — **P2**

- **P2-1.** Сайт не зовёт в Telegram. Нет sticky-баннера, нет всплывашки после скролла, нет inline-CTA в статьях.
- **P2-2.** Telegram не уводит на сайт на что-то большее, чем «прочесть». Нет лид-магнитов («Еженедельная подборка PDF», «Чеклист: как выбрать LLM»).
- **P2-3.** Нет email-capture. Потеря ~30% трафика, который предпочитает email Telegram’у.
- **P2-4.** Нет мест под ваши продукты. Нет страницы «Продукты», «Курсы», «Консультации». Нет нативных блоков в статьях.
- **P2-5.** Нет пикселя Яндекс.Метрики / VK Pixel → нет ретаргета.

### 3.6 Пайплайн, надёжность, масштабирование — **P2**

- **P2-6.** Скоринг `scorer.ts` — плоский и примитивный. Все статьи VentureBeat получают одинаковый score. Нет учёта «интересности» текста.
- **P2-7.** `MIN_SCORE_FOR_CLAUDE = 3` (фактически из кода `pipeline/enricher.ts:32`). Согласно `digest-fix-spec.md:76` — должно быть 3. Надо проверить эффективность: сколько реально получают Claude-обработку.
- **P2-8.** В пайплайне нет шага «кластеризация по теме». Несколько источников про одно событие — разные карточки.
- **P2-9.** Нет А/Б тестирования заголовков/лидов.
- **P2-10.** Нет ручного редактора. Если Claude сгенерировал бред, поправить нельзя — только пересгенерить.
- **P2-11.** Нет системы «закреп» главного материала дня.
- **P2-12.** Нет Playwright-скрапера для сайтов без RSS (Reuters, Anthropic, DeepMind) — теряется топовый контент.

### 3.7 Аналитика и обратная связь — **P2**

- **P2-13.** Нет метрик внутри Telegram (views/link → клик на сайт). Нужна UTM-разметка всех ссылок из бота.
- **P2-14.** Нет ежедневного health-dashboard’а (сколько статей спарсено, сколько опубликовано, какие источники падают).
- **P2-15.** Нет отчётности по Claude — token spend в месяц, % fallback’ов.

## 4. Топ-10 правок в порядке приоритета

Эти десять изменений делают 80% маркетингового эффекта. Остальное — «полировка», важная для второй волны роста.

1. **Переписать промпт Claude и сменить концепт** — вместо `why_it_matters` ввести `lead` (хук) + `summary` (3–5 тезисов) + `card_teaser` (1 строка для карточки). Выкинуть «Почему это важно» с сайта и из Telegram.
2. **Переделать Telegram-формат** — «Заголовок → 1–2 строки сути → CTA», включить превью ссылки, добавить хэштеги.
3. **Редакторский порог публикации** — отсекать сырые DeepL-переводы без редактуры, короткие статьи, пустые лиды.
4. **Новый промпт на полноценную переработку `ru_text`** — не переводить DeepL’ом, а **пересобирать статью** Claude’ом в нашем тоне (вывод: 2–4 абзаца русского текста с лидом, фактами, последствием).
5. **JSON-LD + правильные мета-теги** — `NewsArticle` schema, корректный `description` (не из `why_it_matters`), Open Graph с динамическими картинками.
6. **Герой-блок главной + брендинг дня** — «AI Дайджест / 17 апреля / главное за сутки», слоган, лента выпуска, боковая колонка.
7. **Топик-лендинги с SEO-интро** — для `/russia`, `/topics/ai-labs`, `/topics/ai-industry` написать 400+ слов ручного интро (или генерировать Claude’ом раз в неделю), H1+H2+внутренние ссылки.
8. **Воронка в Telegram из сайта и обратно** — sticky CTA «Подписаться на дайджест», лид-магнит, UTM-метки, email-capture как второй канал.
9. **Кластеризация новостей по смыслу** — чтобы 5 топов в ТГ были **про 5 разных событий**, а не 5 раз про одно.
10. **Breaking-режим** — если появляется статья с очень высоким приоритетом (релиз от OpenAI/Anthropic), выпускать её в ТГ сразу, а не ждать 9:00.

## 5. Пакет правок — детальный разбор под разработку

Каждый пункт самодостаточен. Можно брать в работу отдельно.

### 5.1 [P0] Переписать концепт AI-обогащения

**Файлы:** `pipeline/claude.ts`, `pipeline/enricher.ts`, `supabase/schema.sql`, `lib/supabase.ts` (тип `Article`).

**Проблема.** Один абстрактный `why_it_matters` не решает ни карточку, ни статью, ни TG. Нужны разные форматы под разные точки потребления.

**Решение.** В БД добавить три поля:

```sql
ALTER TABLE articles ADD COLUMN lead TEXT;           -- 1–2 предложения, хук с фактом
ALTER TABLE articles ADD COLUMN summary TEXT;        -- 3–5 буллетов: что произошло, цифры, последствие
ALTER TABLE articles ADD COLUMN card_teaser TEXT;    -- 1 строка для карточки (80–120 знаков)
ALTER TABLE articles ADD COLUMN tg_teaser TEXT;      -- 1–2 строки для TG (140–220 знаков)
ALTER TABLE articles ADD COLUMN editorial_body TEXT; -- финальный редакторский текст статьи на сайте (2–5 абзацев)
ALTER TABLE articles ADD COLUMN quality_ok BOOLEAN DEFAULT false; -- прошла ли материал редакторский порог
```

В `pipeline/claude.ts` — новый промпт, который возвращает JSON с этими полями. Пример промпта (под `claude-haiku-4-5`, structured output):

```
Ты — выпускающий редактор русскоязычного медиа Malakhov AI Дайджест.
Тон — как у «Коммерсантъ» и vc.ru: спокойный, фактологический, без воды и без восторгов.
Никаких «эта новость важна», «важно отметить», «стоит обратить внимание», «в мире ИИ».

На вход дан заголовок и текст статьи. Ты:
1. Придумываешь русскоязычный заголовок (до 90 символов), который точно описывает факт. Без кликбейта, но с конкретикой: цифра, имя, продукт.
2. Пишешь `lead` — 1–2 предложения, которые ОБЪЯСНЯЮТ, что произошло. В первой фразе должен быть главный факт. Никаких «компания X объявила». Сразу суть: «OpenAI закрыла доступ к API GPT-4 для пользователей с бесплатных тарифов — экономия $40 млн/год».
3. Пишешь `summary` — 3–5 буллетов, каждый — конкретный факт/цифра/имя. Не повторять лид.
4. Пишешь `card_teaser` (до 120 знаков) — одна строка, которая висит под заголовком в карточке на сайте. Цель: заставить кликнуть. Не «почему это важно», а «что случилось, что меняется».
5. Пишешь `tg_teaser` (140–220 знаков) — то же, но для Telegram. Допустимо дать один дополнительный факт из тела, который не влез в card_teaser.
6. Пишешь `editorial_body` — 2–4 абзаца связного русского текста. Структура: лид → контекст → детали/цифры → последствие или цитата. 600–1200 знаков. Не копировать исходный текст дословно.
7. Ставишь `quality_ok = true`, только если в материале есть минимум один конкретный факт, имя собственное и цифра/дата/продукт. Иначе false.

Формат ответа — строго JSON:
{
  "ru_title": "...",
  "lead": "...",
  "summary": ["...", "...", "..."],
  "card_teaser": "...",
  "tg_teaser": "...",
  "editorial_body": "...",
  "quality_ok": true
}
```

**Экономика.** Haiku 4.5 — ~$1/1M вход + $5/1M выход. Средняя статья: ~2000 токенов вход, ~600 выход. Ежедневно 20–40 статей. Итого ~$1–2/мес при 30 статьях/день. Допустимо.

**Что меняется в коде:**
- В `claude.ts` заменить `generateWhyItMatters` на `generateEditorial(title, text): Promise<EditorialOutput>`.
- В `enricher.ts` — вместо вызова `translateText` + `translateTitle` + `generateWhyItMatters` сделать **один вызов `generateEditorial`** (DeepL остаётся как fallback для случаев, когда Claude недоступен).
- В `ArticleCard.tsx` — показывать `ru_title` + `card_teaser` вместо `why_it_matters`.
- На странице статьи — показывать `lead` жирным, `editorial_body` основным текстом, `summary` как буллеты в отдельном блоке «Кратко».

**Критерий готовности.**
- На 10 случайных статьях `card_teaser` — понятная русская строка без «важно/нужно/этот материал».
- На странице статьи читатель за 30 секунд понимает суть без перехода в оригинал.

### 5.3 [P0] Редакторский порог `quality_ok`

**Файлы:** `pipeline/enricher.ts`, `lib/articles.ts`.

**Проблема.** Сейчас `published = true` ставится для всех `score >= 2`. Нет отдельного флага «годится для публики».

**Решение.**
- `quality_ok` приходит из Claude (см. 5.1).
- Дополнительные фильтры в `enricher.ts` перед `published = true`:
  - `editorial_body.length >= 300` (иначе: слишком короткая).
  - `lead` не содержит стоп-слов: `важно`, `интересно`, `стоит отметить`, `в мире ИИ`, `эта новость`.

  - `summary.length >= 2` пункта.
- Страницы сайта (`lib/articles.ts`) должны фильтровать по `.eq('quality_ok', true)` **в дополнение** к `.eq('published', true)`.
- Material без `quality_ok` — остаётся в БД для внутренней ценности (аналитика, кластеризация), но не попадает на сайт/в ТГ.

**Критерий готовности.** На сайте нельзя найти статью с машинным заголовком или 100-символьным телом.

### 5.6 [P1] Кластеризация по смыслу

**Файл:** новый `pipeline/cluster.ts`, интеграция в `pipeline/enricher.ts` и `bot/daily-digest.ts`.

**Проблема.** В один день в Telegram могут уехать 2–3 статьи про одно событие (например, разные источники написали про релиз Gemini 3.0).

**Решение (минимальный MVP).**
- После обогащения считать для каждого материала **embedding `ru_title + lead`** через OpenAI `text-embedding-3-small` (~$0.02/1M токенов, копейки).
- Сохранять в поле `embedding VECTOR(1536)` (Supabase поддерживает pgvector).
- При выборке «топ за сутки» для Telegram: брать первую статью с максимальным score, потом каждую следующую добавлять **только если cosine distance > 0.3** от всех уже взятых.
- В итоге — 5 разных тем.

**Критерий.** В день релиза любой нашумевшей модели (Gemini/Claude/GPT) в TG-дайджесте только одна статья про неё, не три.

### 5.8 [P1] Tone of voice и стиль-гайд

**Файл:** `docs/editorial_style_guide.md` (новый).

**Содержание (короткий чек-лист для промпта и ручной правки):**
- Пишем в 3-м лице, настоящем/прошедшем времени.
- Никакого «мы». Только «редакция», «автор».
- Никогда: «в мире ИИ», «стремительное развитие», «революция», «настоящий прорыв», «это важно», «стоит отметить», «как известно».

- Никакого markdown внутри `editorial_body` (он рендерится как plain paragraph).
- Никакого «как сообщает источник» в первом абзаце.

Этот файл — источник истины для Claude-промпта и для ручной редактуры.

## 9. Что НЕ делать сейчас (и почему)

- **Не делать авторские лонгриды.** Это не про MVP. Сначала — поток качественных коротких заметок. Лонгриды — после 3000 подписчиков.
- **Не покупать рекламу до 5.1–5.3.** Платный трафик на сырой контент — слив денег.
- **Не запускать платную подписку.** Рано. Сначала — доверие (3 мес регулярного качества), потом — деньги.
- **Не делать свою CMS.** Supabase + Next.js + ручная админка (5.11) — этого хватит на первые 50 тыс. показов/день.
- **Не интегрировать 10+ источников сразу.** Текущие 14 — более чем достаточно. Добавлять только после кластеризации (чтобы не дублить).
- **Не переходить на Sonnet/Opus для обогащения.** Haiku 4.5 хватит. Sonnet — только для weekly-разборов (если запустим).

2. **Подтвердить tone of voice:** «Коммерсантъ / vc.ru» — спокойный, фактологический, без пафоса.
3. **Проверить доступы:** у Claude есть квота на ~500 вызовов/день? (оценка расхода — $1–2/мес).
4. **Дать доступ к Search Console и Яндекс.Вебмастеру** — нужно для 5.5.
5. **Определить бюджет на старт:** 0 ₽ (органика) или 10–30 тыс. ₽/мес (первая реклама в AI-каналах после недели 2).
6. **Список ваших продуктов** — для страницы `/products` и инлайн-CTA. Хотя бы 2–3 оффера.
7. **1 час на созвон через 2 недели** для оценки результата и корректировки.

## 11. Итог

Текущий продукт — технически работающий RSS-агрегатор. Чтобы он стал **медиа, которое ждут**, нужно **4 недели сфокусированной работы** по плану выше. Ключевое — не добавление фич, а **смена концепта «почему это важно» → «что произошло и что меняется»**. Всё остальное — инструменты для доставки этого контента.

Когда будете готовы — начинаем с блока **5.1 (Claude-промпт)** + блока **5.2 (Telegram-формат)**. За 3–4 рабочих дня вы увидите кардинально другой сайт и другой канал.
