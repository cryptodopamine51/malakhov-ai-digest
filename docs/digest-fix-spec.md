# Задача: Полная доработка пайплайна дайджеста

## Цель

Пайплайн должен работать так:
1. Каждые 30 мин → новые статьи парсятся и падают в базу
2. Каждые 2 ч → статьи обогащаются: очищаются, переводятся, получают редакторский вывод от Claude, публикуются на сайте
3. В 09:00 МСК → из опубликованных за вчера выбираются лучшие, их страницы проверяются, дайджест уходит в ТГ-канал

Результат в ТГ — не просто список ссылок, а редакторски оформленный дайджест с объяснением почему каждая новость важна.

---

## Блок 1 — Очистка текста (fetcher.ts)

### Проблема
Habr и некоторые другие источники отдают в начале страницы UI-метадату, которая попадает в текст:
```
"Уровень сложностиСреднийВремя на прочтение4 минОхват и читатели2.5K..."
```
Эти строки уходят в `ru_text` и как превью в ТГ.

### Что сделать
В `pipeline/fetcher.ts` после получения текста через Readability добавить постобработку:

**Паттерны для удаления (regex):**
```
/Уровень сложности\w+/g
/Время на прочтение\d+\s*мин/g
/Охват и читатели[\d.,]+[KkМм]?/g
/Читать далее/g
/Поделиться/g
```

**Общая логика:**
- Убирать строки короче 40 символов в начале текста (это UI-элементы, не контент)
- Убирать строки целиком состоящие из заглавных букв (кнопки, метки)
- Обрезать текст до первого осмысленного абзаца (длина > 100 символов)

### Критерий готовности
Первые 200 символов `ru_text` для любой статьи с Habra — читаемый русский текст без служебных слов.

## Блок 2 — Скоринг (scorer.ts)

### Проблема
- `MIN_SCORE_FOR_CLAUDE = 4`, но большинство статей набирают максимум 3
- Все статьи выходят с одним и тем же score → нет реального ранжирования
- Статьи из AI-лабораторий (лучший контент) часто не попадают из-за таймаутов фидов

### Что изменить

**1. Снизить порог Claude:** `MIN_SCORE_FOR_CLAUDE = 4` → `MIN_SCORE_FOR_CLAUDE = 3`
Это значит Claude будет вызван для всех публикуемых статей (MIN_SCORE_TO_PUBLISH = 2).
Для экономии — вызывать только если `score >= 3`, а не для всех с score 2.

**2. Расширить скоринг:**

```ts
// Добавить к существующим правилам:

// +1 за наличие cover_image (статьи с картинкой обычно качественнее)
if (article.cover_image_url) score += 1

// +1 за длинный текст (> 1000 символов после очистки — статья, а не заметка)
if ((article.original_text?.length ?? 0) > 1000) score += 1

// +1 за русскоязычный источник (приоритет для русской аудитории)
if (article.source_lang === 'ru') score += 1
```

**3. Обновить константы:**
```ts
const MIN_SCORE_TO_PUBLISH = 2   // без изменений
const MIN_SCORE_FOR_CLAUDE = 3   // было 4
```

### Критерий готовности
После прогона enricher в логах хотя бы 30% статей должны иметь пометку `Claude ✓`.
Статьи из одного источника за один день имеют разные score (нет flat-ранжирования).

## Блок 3 — Генерация why_it_matters (claude.ts)

### Проблема
`why_it_matters` не генерируется → в ТГ уходит сырой обрезанный текст вместо редакторского вывода.

### Что проверить в claude.ts
Открыть `pipeline/claude.ts` и убедиться что:
1. Промпт просит Claude написать 1-2 предложения на русском: **почему эта новость важна для AI-аудитории**
2. Промпт не просит пересказ — только вывод и значимость
3. Модель — `claude-haiku-4-5` (дёшево, быстро)
4. Если Claude вернул пустую строку или ошибку — fallback на первые 2 предложения из `ru_text`

**Правильный промпт:**
```
Ты — редактор русскоязычного AI-дайджеста. Напиши 1-2 предложения: почему эта новость важна 
для людей, следящих за AI-индустрией? Что она меняет или о чём предупреждает? 
Без пересказа, только значимость и вывод. На русском языке.

Заголовок: {ru_title}
Текст: {первые 800 символов ru_text}
```

### Критерий готовности
Для каждой опубликованной статьи с score ≥ 3 поле `why_it_matters` заполнено осмысленным текстом на русском, 1-2 предложения, без "Эта статья рассказывает о...".

## Блок 4 — Фиды и таймауты (rss-parser.ts)

### Проблема
Регулярно падают фиды с лучшим контентом:
```
Ошибка фида "VentureBeat AI": Request timed out after 10000ms
Ошибка фида "OpenAI News": Request timed out after 10000ms
Ошибка фида "Hugging Face Blog": Request timed out after 10000ms
Ошибка фида "Google Research Blog": Request timed out after 10000ms
```

### Что сделать
В `pipeline/rss-parser.ts`:

**1. Увеличить таймаут** с 10000ms до 20000ms для всех фидов.

**2. Добавить 1 retry с паузой:**
```ts
async function fetchFeedWithRetry(url: string, timeout = 20000, retries = 1): Promise<...> {
  for (let attempt = 0; attempt <= retries; attempt++) {
    try {
      return await fetchFeed(url, timeout)
    } catch (err) {
      if (attempt === retries) throw err
      await sleep(3000)
    }
  }
}
```

**3. Параллельная загрузка фидов с ограничением:**
Сейчас фиды грузятся последовательно. Добавить параллельную загрузку батчами по 5.

### Критерий готовности
В логах ingest не более 2 ошибок таймаута за запуск (из 16 фидов).
VentureBeat, OpenAI, Hugging Face, Google — успешно парсятся в 8 из 10 запусков.

## Блок 5 — Формат дайджеста в ТГ (daily-digest.ts)

### Проблема
Текущий формат:
```
*1. Заголовок статьи*
Уровень сложностиСреднийВремя на прочтение4 мин...  ← мусор
[Читать →](ссылка)
```

Нужный формат:
```
*1. Заголовок статьи*
💡 Почему важно: Claude-вывод про значимость новости
[Читать →](ссылка)
```

### Что изменить в buildDigestText()

```ts
// Было:
const body =
  article.why_it_matters
    ? escapeMd(article.why_it_matters)
    : escapeMd((article.ru_text ?? article.original_text ?? '').slice(0, 120))

// Стало:
const body = article.why_it_matters
  ? `💡 ${escapeMd(article.why_it_matters)}`
  : null  // если нет why_it_matters — не показываем мусорный снипет
```

**Также улучшить структуру поста:**
```
🤖 *AI Дайджест от Malakhov AI* — {дата}
Топ новостей за {вчерашний день} по версии редакции

*1. {заголовок}*
💡 {why_it_matters}
[Читать на сайте →]({url})

*2. ...*

——————
📰 Все новости: {siteUrl}
📢 Канал: @malakhovaidigest
```

### Критерий готовности
- В каждом посте у каждой статьи есть блок `💡 Почему важно:`
- Нет сырых HTML-фрагментов или Habr-метадаты в тексте
- Структура поста читается как редакторский дайджест, а не список ссылок

## Блок 6 — Защита от двойной отправки

### Проблема
Дайджест может уйти дважды: GitHub Actions в 09:00 + ручной запуск.

### Что сделать
Добавить в `daily-digest.ts` проверку: если сегодня дайджест уже отправлялся — выходить.

```ts
// Проверяем есть ли tg_sent = true за последние 8 часов
const { count } = await supabase
  .from('articles')
  .select('*', { count: 'exact', head: true })
  .eq('tg_sent', true)
  .gte('updated_at', new Date(Date.now() - 8 * 60 * 60 * 1000).toISOString())

if ((count ?? 0) > 0) {
  log('Дайджест за сегодня уже отправлен — пропускаем')
  process.exit(0)
}
```

### Критерий готовности
При двух последовательных запусках `npm run tg-digest` второй завершается с "уже отправлен" без дублирования.

## Блок 7 — Окно выборки статей для дайджеста

### Проблема
Сейчас `tg-digest.ts` берёт статьи за последние 24 часа. Если cron запускается в 09:00, берутся статьи с 09:00 вчера — но самые ранние утренние новости (08:00-09:00) могут не попасть, а вчерашние вечерние (21:00-23:00) — попасть.

### Что сделать
Изменить логику выборки: брать статьи за **вчерашний календарный день** (00:00 - 23:59 МСК), а не последние 24 часа.

```ts
// Определяем вчерашний день по МСК (UTC+3)
const now = new Date()
const moscowOffset = 3 * 60 * 60 * 1000
const moscowNow = new Date(now.getTime() + moscowOffset)
const yesterday = new Date(moscowNow)
yesterday.setUTCDate(yesterday.getUTCDate() - 1)

const from = new Date(Date.UTC(yesterday.getUTCFullYear(), yesterday.getUTCMonth(), yesterday.getUTCDate(), 0, 0, 0) - moscowOffset)
const to   = new Date(Date.UTC(yesterday.getUTCFullYear(), yesterday.getUTCMonth(), yesterday.getUTCDate(), 23, 59, 59) - moscowOffset)
```

### Критерий готовности
Дайджест всегда содержит новости ровно за вчерашний день по московскому времени, независимо от времени запуска.

## Итоговый чеклист проверки перед запуском

### Проверка обогащения (после npm run enrich)
- [ ] В логах нет строк "Уровень сложности" — текст чистый
- [ ] Хотя бы 3 из 5 публикаций имеют пометку `Claude ✓`
- [ ] Ни одна статья не имеет `why_it_matters = null` при score ≥ 3
- [ ] Статьи из VentureBeat / OpenAI / Hugging Face присутствуют в базе

### Проверка сайта (после деплоя / ISR)
- [ ] `curl -o /dev/null -w "%{http_code}" https://news.malakhovai.ru/articles/{slug}` → 200
- [ ] Главная страница отдаёт не менее 5 карточек
- [ ] Страница `/russia` содержит статьи с Habra/vc.ru
- [ ] Блок "Почему важно" отображается на странице статьи (если есть why_it_matters)

### Проверка дайджеста (npm run tg-digest)
- [ ] В логах нет "Страница недоступна, пропускаем"
- [ ] В логах нет "Уровень сложности" в теле сообщения
- [ ] Сообщение содержит блок `💡` у каждой статьи
- [ ] Повторный запуск → "Дайджест за сегодня уже отправлен"
- [ ] Ссылки в ТГ ведут на реальные страницы (HTTP 200)

### Проверка фидов (npm run ingest)
- [ ] Не более 2 таймаутов из 16 фидов
- [ ] VentureBeat, OpenAI, Hugging Face — в списке успешно обработанных
- [ ] Количество добавленных статей за запуск: от 5 до 30
