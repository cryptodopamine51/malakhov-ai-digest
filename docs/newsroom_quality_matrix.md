## 1. Главная Страница

### Автоматические проверки

- hero-блок рендерится
- есть главное событие
- есть реальные карточки событий
- метрики не показываются как нулевые, если есть контент
- не рендерятся пустые блоки разделов
- в HTML нет `Событие в AI`
- в HTML нет fallback-формулировок

### Ручная приемка

- первый экран выглядит как медиа
- глаз сразу цепляется за главный материал
- боковая колонка помогает навигации, а не перегружает
- дизайн соответствует визуальному референсу: светлый, чистый, современный

## 2. Страница Выпуска

### Автоматические проверки

- есть issue intro
- есть реальные section main cards
- у main card не абстрактный текст про раздел, а реальный материал
- в secondary grid нет заглушек
- есть корректные ссылки на разделы и события
- issue main card содержит 2–3 компактных редакционных абзаца

### Ручная приемка

- страница воспринимается как номер редакции
- главный материал раздела визуально выделен
- выпуск легко просматривать сверху вниз

## 3. Страница События

### Автоматические проверки

- есть основной заголовок
- есть article body из связанных абзацев
- нет блоков `Почему это важно`, `Что это меняет`, `Кто выигрывает / проигрывает`
- нет повторов заголовка в первом абзаце
- нет служебных фраз про число источников
- нет лишнего английского языка
- есть связанные материалы и навигация по выпуску

### Ручная приемка

- статья читается как заметка
- первый абзац держит факт
- середина дает детали и контекст
- финал объясняет, что меняется дальше
- сайдбар не отвлекает от основного текста

## 4. Карточки Событий

### Автоматические проверки

- есть нормальный заголовок
- нет `Событие в AI`
- нет source leakage
- нет дублирования заголовка в summary
- нет мусорных placeholder-формулировок

### Ручная приемка

- карточка читается за 2–3 секунды
- заголовок и описание дают понятный сигнал о событии
- карточки выглядят как единая система

## 5. Редакционный Текст

### Автоматические проверки

- статья не повторяет заголовок
- статья не содержит запрещенные шаблоны
- статья не содержит лишние английские слова
- D2 остается короткой и плотной
- D3 содержит больше деталей, чем D2
- D4 включается только на богатом source pack
- цитаты используются только при пользе
- числа нормализуются по-русски

### Ручная приемка

- текст не выглядит машинным
- текст не выглядит копией источника
- текст не распадается на бессвязные абзацы

## 6. Поисковая Часть

### Автоматические проверки

- у страницы есть корректный `<title>`
- у страницы есть корректный `description`
- есть `canonical`
- есть корректные Open Graph поля
- есть структурированные данные, когда они внедрены

### Ручная приемка

- сниппет страницы выглядит прилично
- мета-описание хочется открыть в выдаче

## 7. Русский Язык

### Автоматические проверки

- нет лишнего английского в публичном HTML
- нет технических англоязычных формулировок
- бренды и модели не ломаются при русификации

### Ручная приемка

- сайт понятен русскоязычному читателю без технического контекста

## 8. Изображения

### Автоматические проверки

- у главных материалов есть изображение или обложка
- карточка и страница умеют рендерить изображение
- fallback-режим изображений стабилен

### Ручная приемка

- изображения не выглядят случайными
- изображения поддерживают общий стиль медиа

## 9. Запреты Для Публичного Слоя

Это должно быть запрещено тестами или явными проверками:

- `Событие в AI`
- `Сегодня день спокойный`
- служебные формулировки про источники
- повтор заголовка в lede
- сильный англоязычный мусор
- пустые карточки
- пустые разделы
- section main card без реального события

## 10. Финальная Приемка Продукта

Продукт принимается как рабочий, если одновременно выполняются условия:

- главная выглядит живой и сильной
- выпуск выглядит как номер редакции
- страница события выглядит как полноценная статья
- русский язык чистый
- слабые материалы не доходят до публикации
- поисковая структура корректна
- визуальный слой сайта цельный и убедительный
