# Редакционный стайл-гайд — Malakhov AI Дайджест

**Версия:** 2026-04-26. Этот файл — источник истины. Подгружается в системный промпт Claude.

---

## Тон

Спокойный, фактологический деловой журналист. Референс: «Коммерсантъ», vc.ru, habr editorial.
Без пафоса, без агитации, без маркетинговой риторики.

---

## Запрещённые слова и обороты

- **Вводные пустышки:** «важно отметить», «стоит обратить внимание», «как известно», «что ж», «итак», «действительно».
- **Маркетинговый пафос:** «революция», «прорыв», «настоящий», «поистине», «невероятный», «потрясающий», «впервые в истории», «это меняет всё», «эта новость важна».
- **Канцеляризмы:** «осуществлять», «в рамках», «что касается», «на сегодняшний день».
- **Контекстные клише:** «в мире ИИ», «эта новость», «как сообщает источник».

---

## Правила написания

1. Первое предложение лида всегда содержит минимум один конкретный якорь: имя собственное / число / дату / название продукта.
2. Первый абзац никогда не повторяет заголовок дословно.
3. Никогда не начинать с «Компания X объявила» / «Как сообщает источник».
4. «AI» пишем «ИИ», кроме составных названий продуктов (AI Studio, Gemini AI).
5. Названия моделей — в оригинале: GPT-4o, Claude 3.5 Sonnet, Gemini 2.5 Pro, Llama 3.1, YandexGPT 4, GigaChat.
6. Английские термины только если нет русского аналога: LLM, API, open-source, benchmark. «Эмбеддинг» — писать «эмбеддинг», не «embedding».
7. Опираться только на факты из источника. Запрещено добавлять цифры, имена, продукты, которых в источнике нет.

---

## Однозначность смысла в первом абзаце

Первый абзац обязан однозначно отвечать на вопрос «что произошло?».

Запрещены конструкции, где глагол допускает противоположные толкования:

- «закрыл фонд» — двусмысленно: прекратил работу или полностью собрал фонд. Использовать:
  «привлёк $X в новый фонд», «полностью собрал фонд на $X», «прекратил работу фонда».
- «остановил продукт» — двусмысленно: временно заморозил или окончательно прекратил. Уточнять.
- «вышел из X» — двусмысленно: запустил продукт или покинул рынок/компанию. Уточнять.

Если оригинальный заголовок содержит такую двусмысленность, `ru_title` обязан её снять, а не
сохранять оригинал дословно. В первом предложении лида должен быть конкретный субъект и действие:
«Кто что сделал». «X закрыл фонд» — недостаточно. «X собрал крипто-фонд на $2.2B, пятый по счёту» —
достаточно.

## Структура редакторского текста (editorial_body)

- **Абзац 1:** лид-развёртка. Что произошло, с конкретикой.
- **Абзац 2:** контекст. Почему это появилось сейчас.
- **Абзац 3:** детали. Цифры, имена, цитаты, сравнения.
- **Абзац 4:** последствия. Кого касается, на что влияет.
- **Абзацы 5–7 (опционально):** расширение при наличии материала.

Абзацы разделяются одной пустой строкой.

Обычно тело идёт без markdown-списков, заголовков и жирного текста. Исключение — материалы,
которые по сути являются перечислением: «5 трендов», «10 способов», «N компаний». В таких текстах
цифры и пункты должны быть явно выделены нумерованным markdown-списком или короткими подзаголовками
с номерами.

## Таблицы и структурированные данные

`article_tables` используются только там, где в источнике есть структурированные данные:

- сравнения: продукт X vs продукт Y;
- перечисления с одинаковыми атрибутами: компания, раунд, сумма, инвесторы;
- временные линии: дата или год + событие;
- метрики: benchmark, score, модель, параметр.

Таблицу нельзя натягивать из обычного текста. Если факты не образуют повторяемую структуру,
`article_tables` остаётся пустым.

## Research

Для `ai-research` материал должен быть глубже обычной новости. В user message Claude получает
явный блок `RESEARCH:` с требованием раскрыть: проблему, подход, результаты, ограничения,
значение для отрасли.

Минимальный размер `editorial_body` — 1500 символов. Это требование прописано:
- в системном промпте (КРИТЕРИЙ quality_ok);
- в category hint user message;
- в gate collect (`research_too_short` если < 1500).

Короткие research-материалы отклоняются на стадии collect.

---

## Startups

Для `ai-startups` Claude получает явный блок `STARTUPS:` в user message с требованием указать
при наличии данных в источнике:

- размер раунда;
- инвесторов;
- оценку;
- продукт компании одной фразой;
- отличие от конкурентов.

Если этих данных нет в источнике, их нельзя додумывать.

---

## Поля и лимиты

| Поле | Объём |
|---|---|
| `ru_title` | до 90 символов, без точки в конце |
| `lead` | 200–360 символов, 1–2 предложения |
| `summary` | 3–5 буллетов по 60–180 символов |
| `card_teaser` | 80–140 символов, для карточки |
| `tg_teaser` | 160–260 символов, для Telegram |
| `editorial_body` | 1200–5000 символов, 3–7 абзацев; для `ai-research` минимум 1500 |
| `article_tables` | 0–3 таблицы только при наличии структурированных данных |

## Evergreen quality bar

Для evergreen-гайдов планка выше, чем для news. См. подробности в
`docs/editorial/seo-article-publication-standard.md` (раздел «Evergreen quality bar»). Главное:

- **Lead anchor.** Первое предложение лида содержит число / дату / имя собственное / источник.
- **`verifiedAt`.** В шапке гайда render «Актуальность проверена: <дата>»; поле обязательное в `content/guides/meta/<slug>.json`.
- **Numerical worked example.** Для числовых intents (стоимость / ROI / окупаемость / сравнения) — минимум один развёрнутый расчёт по шаблону «Ситуация → данные → формула → результат → выводы», не таблица категорий.
- **Кейс.** Минимум один H2/H3 «Кейс / Сценарий / Ситуация / Мини-кейс». Иерархия: реальный публичный → анонимизированный → редакционный сценарий. В опубликованном тексте не ставим служебные пометки вроде «Редакционный пример» и не выдаём обезличенный сценарий за наш/клиентский кейс без основания.
- **Counter-strategy.** Отдельный H2 «Когда не стоит / не окупится / Ошибки внедрения» с 3–5 конкретными критериями.
- **Российский контекст.** 152-ФЗ для данных/клиентов/HR; GigaChat / YandexGPT для тарифов; Яндекс.Директ / ВКонтакте — для маркетинга.
- **Internal linking.** ≥ 2 inline-ссылок в теле на `/guides`, `/categories/<cat>`, `/russia` (не считая related). Ссылки на ещё не опубликованные гайды запрещены.
- **CTA cap.** ≤ 2 inline-CTA в теле + 1 final-CTA блок с 3 карточками. CTA указывают только на реально существующие поверхности: Telegram-дайджест (`@malakhovaidigest`), форма заявки (`malakhovai.ru/contacts`), личный Telegram (`@iddopamine`). Запрещено обещать чеклисты, PDF, гайды на почту и любые lead-magnet артефакты, которых нет. Дефолтный final-CTA набор — `DEFAULT_FINAL_CTA_CARDS` в `app/guides/[slug]/page.tsx`.
- **Indexation.** `noindex: true` ставится только пока нет cover; после готовности cover сразу `noindex: false` + IndexNow. Никакого 3–7-дневного review-окна.
- **Российские источники.** White-list 8 источников (Tier 1: Яков и Партнёры, НИУ ВШЭ ИСИЭЗ, TAdviser, CNews Analytics; Tier 2: Sber/SberAI, Yandex Research; Tier 3: Forbes Russia, Ведомости.Технологии) — подробности в `docs/editorial/seo-article-publication-standard.md` §12. Список внутренний, публично не enumerate'им.
- **Картинки.** Cover ≥ 80 KB; готовятся ТОЛЬКО через подписку ChatGPT (Plus / Pro / Codex), не через image API. Локальные SVG-схемы допустимы для inline-диаграмм.
- **Запрещённые приёмы.** Инфобиз-клише, ссылки на будущие гайды, FAQPage без visible FAQ, дублирующее markdown TOC при наличии sticky aside.

Эти правила проверяются `npm run evergreen:check -- --slug=<slug>`. Errors блокируют публикацию, warnings — нет, но их видит редактор.
