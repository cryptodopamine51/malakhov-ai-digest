# Спека: backfill разделов Лаборатории, Инвестиции, Стартапы

## Проблема

Разделы `Лаборатории`, `Инвестиции`, `Стартапы` выглядят пустыми или слабыми по наполнению. Для editorial-продукта это бьёт и по доверию, и по retention.

## Цель

Подготовить и опубликовать в каждый из разделов минимум 7–10 качественных материалов, включая статьи за прошлые даты, если они релевантны и соответствуют редакционному уровню.

## Скоуп

**Разделы:**
- `/topics/ai-labs`
- `/topics/ai-investments`
- `/topics/ai-startups`

**Источники данных:**
- уже существующие статьи в базе;
- новые материалы, полученные после расширения source set;
- допускается переобогащение уже сохранённых raw items / unpublished items за прошлые даты.

## Что сделать

### 1. Аудит текущей базы

- Посчитать, сколько материалов уже есть по трём топикам:
  - `published=true`
  - `quality_ok=true`
- Отдельно выявить:
  - статьи без нужного topic, но тематически подходящие;
  - статьи `published=false`, которые можно реанимировать;
  - raw / enriched записи за прошлые даты, которые можно допереработать.

### 2. Правила backfill

- Разрешить публикацию качественных материалов за прошлые даты, если:
  - они не устарели содержательно;
  - материал остаётся ценным как контекст/разбор/релиз;
  - не выглядит как “мёртвый архивный мусор”.
- Для старых материалов не искажать дату события.
- Если backfill делается вручную или полуавтоматически, фиксировать источник отбора и причину публикации.

### 3. Проверить topic assignment

- Уточнить правила классификации для:
  - `ai-labs`
  - `ai-investments`
  - `ai-startups`
- Добавить/обновить тесты на граничные случаи:
  - инвестиционный раунд AI-стартапа;
  - релиз от OpenAI/Anthropic/Google DeepMind;
  - материал о стартапе без явного funding event.

### 4. Наполнить разделы

- Довести каждый раздел до 7–10 материалов `published=true, quality_ok=true`.
- Перепроверить первые 3–5 карточек каждой рубрики вручную:
  - заголовки;
  - topics;
  - cover;
  - editorial body.

## Тесты

### SQL / data checks

- Запрос по каждому topic показывает минимум 7 записей:
  - `published = true`
  - `quality_ok = true`
- Нет дублей по `original_url` / `slug`.

### Кодовые

- Тесты классификации тем на граничных кейсах.
- Тесты/скрипт re-enrichment для backfill, если меняется pipeline.

### Ручные

- Открыть 3 раздела на production.
- Убедиться, что:
  - featured-статья есть;
  - сетка заполнена;
  - контент выглядит тематически однородно;
  - нет мусорных или случайно попавших статей.

## Критерии приёмки

- В `Лаборатории`, `Инвестиции`, `Стартапы` доступно минимум по 7 материалов.
- Материалы действительно соответствуют рубрике, а не просто искусственно допихнуты.
- Публикация старых материалов не ломает сортировку и не создаёт misleading “сегодняшнее” впечатление.
