# ChatGPT Image Prompts — Как выбрать первый ИИ-проект в бизнесе

> **Что это.** Готовое сообщение для ChatGPT в формате scenario cards. По нему нужно сгенерировать 4 картинки для статьи `/guides/kak-vybrat-pervyj-ii-proekt-v-biznese`.
>
> **Как пользоваться.**
>
> 1. Открой ChatGPT Plus / Pro / Codex.
> 2. Не отправляй все карточки одним сообщением: ChatGPT может собрать коллаж. Для каждой генерации копируй `ОБЩИЙ СТИЛЬ` + одну карточку конкретной картинки.
> 3. Вставь в чат общий стиль + `КАРТИНКА 1`, затем общий стиль + `КАРТИНКА 2` и так далее.
> 4. PNG клади в `content/evergreen/packages/kak-vybrat-pervyj-ii-proekt-v-biznese/raw-images/`. Имя файла лучше ставить ровно как в `Save As`, но если ChatGPT скачал файл со случайным именем, это допустимо: `images:prep` сопоставит PNG со слотами по порядку. Если PNG случайно окажутся в корне пакета, скрипт подхватит их, когда `raw-images/` пустой.
> 5. Когда все PNG лежат в `raw-images/`, скажи агенту «картинки положил, продолжай».
>
> **Filename convention.** Все `Save As` ниже уже используют SEO-имена `<slug-short>-<section-keyword>.png`; финальные WebP получат такие же stem'ы.

=== ОБЩИЙ СТИЛЬ ДЛЯ КАЖДОГО ПРОМПТА ===

Привет. Мне нужно сгенерировать иллюстрации для редакционной статьи «Как выбрать первый ИИ-проект в бизнесе: скоринг и 12 кандидатов».

Цель картинок: не декоративные «ИИ-иллюстрации», а понятные editorial visuals. По каждой картинке читатель должен понимать тему раздела: выбор первого ИИ-проекта, скоринг, сравнение кандидатов, решение запускать/усиливать/откладывать.

Единый стиль для всех картинок:

- realistic business-editorial + tactile infographic objects;
- спокойный свет, off-white фон, graphite/deep navy база, один тёплый акцент amber/terracotta;
- реальные рабочие люди допустимы и желательны, но только в конкретном действии;
- если в кадре есть люди: локальная российская деловая аудитория, преимущественно славянская/восточноевропейская внешность, естественная одежда и возрастной разброс; без международного stock-photo casting и diversity-poster эстетики;
- без постановочного stock office, без «люди в костюмах вокруг ноутбука»;
- никаких роботов, светящегося мозга, неоновых элементов, рукопожатий человек+робот;
- cover без читаемого текста, цифр и лейблов;
- inline-картинки желательно без текста; если всё же появятся короткие подписи, только один чистый sans-serif, одинаковая толщина, одинаковая сетка, без смешения шрифтов и без ошибок;
- никаких watermark, логотипов, fake dashboards с бессмысленными надписями;
- каждая картинка должна иметь один главный объект и проходить topic clarity test: её нельзя безболезненно поставить к любой другой статье про ИИ.

Генерируй картинки **по одной за раз**, в порядке ниже. После каждой я скажу «следующая» — тогда переходи к следующей.

Create exactly ONE standalone image for this specific scene only.
Do not include other images from the series.
Do not create a collage, contact sheet, storyboard, grid, or multi-panel overview.

Без заголовка должно быть понятно: команда выбирает первый проект через скоринг и сравнение вариантов.
