Возьми черновик статьи ниже и доведи его до уровня готовой публикации для news.malakhovai.ru.

Задача:
сделать статью экспертной, полезной, SEO-сильной, аккуратно структурированной и готовой к передаче в Codex для публикации.

Проверь и улучши:

1. Редактура:
— убери воду;
— усили тезисы;
— сделай текст более живым и экспертным;
— добавь недостающие объяснения;
— убери повторы;
— сделай переходы между разделами логичными;
— сохрани плотный деловой стиль.

2. Экспертность:
— добавь практические выводы;
— добавь бизнес-сценарии;
— добавь риски;
— добавь ошибки внедрения;
— добавь блок «Вывод Malakhov AI»;
— проверь, что статья полезна предпринимателю, руководителю или маркетологу.

3. SEO:
— проверь title, description, H1;
— предложи slug;
— проверь структуру H2/H3;
— добавь FAQ;
— предложи внутренние ссылки;
— предложи связанные статьи;
— предложи сниппетные блоки: определения, списки, таблицы.

4. Публикация:
— подготовь финальную статью;
— подготовь ТЗ на главное изображение;
— подготовь ТЗ на изображения внутри статьи;
— подготовь alt-тексты;
— подготовь ТЗ для Codex;
— подготовь checklist перед публикацией.

Важно:
не меняй смысл в сторону общих рассуждений. Статья должна стать более конкретной, применимой и сильной.

Чек-лист редактора (Evergreen Quality Bar 2026-05-21)

Финальная редактура обязана подтвердить:

1. Первое предложение лида содержит число / имя собственное / дату / источник (factual anchor).
2. В шапке статьи render видимая дата проверки актуальности — «Актуальность проверена: <дата>», в metadata есть `verifiedAt`.
3. Для числовых intents (стоимость, ROI, окупаемость, метрики, сравнения) — есть хотя бы один развёрнутый расчёт по шаблону «Ситуация → данные → формула → результат → выводы». Расчёт не подменён таблицей категорий.
4. Есть кейс с конкретикой по структуре Ситуация → Что делает ИИ → Что нужно для пилота → Метрики → Итог. Источник: реальный публичный → анонимизированный → редакционный сценарий. В опубликованном тексте не писать «Редакционный пример», не добавлять source-note под кейсом и не выдавать обезличенный сценарий за наш/клиентский кейс без основания.
5. Есть counter-strategy блок «Когда не стоит / не окупится» с 3–5 практическими критериями.
6. Российский контекст и регуляторика учтены: 152-ФЗ упомянут для статей о данных/клиентах/HR; GigaChat/YandexGPT — как локальные альтернативы для статей про тарифы; Яндекс.Директ/ВКонтакте — для статей про маркетинг.
